# CaveWhere 2026.4.2 Release Notes

**CaveWhere 2026.4.2** is a patch release focused on fixing sync, save, and remote-project issues, plus several improvements to how online projects are opened and shared.

---

## New Features

- **Open from Link accepts git clone URLs** — the File → Open from Link… dialog (formerly "Open Shared Link…") now accepts HTTPS, SSH, and bare host/path git clone URLs in addition to CaveWhere share links. The GitHub account picker is hidden automatically for SSH URLs since SSH uses local keys (#426)
- **File → Open from Online…** — a new top-level menu item (Ctrl+Shift+O) surfaces the remote repository workflow directly from the File menu, making online projects easier to discover (#422)
- **Share menu always enabled** — the Share menu item is no longer grayed out for bundled `.cw` projects or projects without a configured remote. The Share dialog itself now handles all cases, including an "Add a remote repository to enable sharing" link (#412)
- **Ellipsis on dialog menu items** — File menu items that open dialogs now show an ellipsis, matching platform conventions (#425)
- **Bold row for checked-out commit in Git history** — the author, SHA, and date labels on the HEAD commit row now all render bold, making the currently checked-out commit easier to spot. The Git history page also defaults to splitting the view in half instead of a fixed 400px pane (#431)
- **More helpful cancellation messages** — save and load operations now propagate the actual cancellation reason (e.g. project closed, sync failed) instead of the generic "Operation canceled."

---

## Bug Fixes

### Sync and Project Loading

- **Fix cave page going blank after sync** — when a sync brought in a peer-added trip, the currently viewed cave page would render blank because the page view was never told to re-resolve its content. Adds regression test coverage via `tst_CaveSync.qml` (#429)
- **Fix sync retry losing pulled remote changes during local churn** — when a sync attempt's reconcile failed due to a stale plan epoch (local model mutated after plan capture), the retry recaptured `beforeHead` after the prior pull's merge commit. This produced an empty diff and caused the reconcile to treat the sync as a no-op, leaving remote changes merged on disk but never applied to the in-memory model. The pre-pull HEAD is now preserved across retries so every reconcile sees the full accumulated window of remote changes
- **Fix discard error after opening a v6 SQLite `.cw` bundle** — v6 conversion created a git repository without an initial commit, so `discardChanges()` failed with "revspec 'HEAD' not found". Conversion now creates the initial commit, matching save-as behavior (#418)
- **Fix duplicate sync job in the task bar** — two "Syncing" entries appeared because both `cwSaveLoad::sync()` and `cwProject::beginSyncOperation()` registered a job. The redundant lower-level registration was removed (#411)

### Saving

- **Fix trip image loss when saving a temporary project as a `.cw` bundle** — saving a new (temporary) project as a bundled `.cw` file left previously imported note images orphaned under the old data root, causing `cwImageProvider` to fail with "device not open". The on-disk data-root rename now runs synchronously before the bundle is packaged (#428)
- **Fix temp directory cleanup** — CaveWhere was leaving temporary directories behind for new projects, bundled `.cw` extraction, and legacy SQLite conversion. Temp directories are now cleaned up automatically via project lifecycle, with safety guards to prevent deletion outside the system temp path

### Remote Projects and Deep Links

- **Fix first-clone auth error on deep links** — the GitHub keychain token is now loaded when the deep link dialog opens, so the first clone no longer fires with an empty token
- **Fix share links with trailing slashes** — URLs like `https://cavewhere.com/open/?repo=...` (as produced by WordPress) are now accepted alongside the untrailing-slash form (#423)

### Stability

- **Fix use-after-free when removing an empty cave** — `cwLinePlotLabelView::removeCaves()` called `deleteLater()` on the removed cave's label group but never took it out of `CaveLabelGroups`. The next line-plot update could then dereference the freed group through a surviving cave's new index. Caught under AddressSanitizer (#434)
- **Fix race in `cwDiskCacher::insert`** — a TOCTOU check around `mkpath()` allowed concurrent threads to race on directory creation, causing intermittent insert failures under load. Qt's `mkpath()` already handles the existing-directory case, so the pre-check was removed

### Platform

- **Fix dark mode on Windows and Linux** — Qt's native Windows Vista style ignored the dark palette; CaveWhere now uses the Fusion style on Windows and Linux, which respects dark mode correctly. macOS continues to use the native style. Also fixes a Theme divider color that had been accidentally hardcoded to a dark value (#439)
- **Fix Windows installer filename capitalization** — the installer is now named `CaveWhere 2026.4.2 64bit` instead of `Cavewhere 2026.4.2 64bit` (#424)
- **Fix Windows build** — renamed `Job::Action::CopyFile` to `Copy` to avoid a collision with the `CopyFile` macro defined in `<windows.h>`, plus follow-up Windows build fixes

## Platform and Build

- **PDF support in Linux and Windows builds** — the CI builds for Linux and Windows now ship with Qt PDF support enabled, matching the macOS build (#413)
- **Linux ARM64 CI builds** — CI now produces Linux ARM64 binaries using the `ubuntu-24.04-arm` GitHub-hosted runner, with arch-specific Conan cache keys to keep the x86_64 and ARM64 caches from colliding
- **Test suite stability** — numerous QML and C++ test fixes to reduce flakiness under nightly/parallel CI load: carpet-mode click retry, skipping empty notes during lead scans, 10s timeouts on page-transition waits, a Windows-only Catch2 DLL fix for `cwSyncChurnFixture`, and a `cwTaskManagerModel` fix that replaces `QSignalMapper` with direct typed connections to avoid a null-`sender()` drop on Windows. Also extracts the multi-peer sync fixture and churn loop into reusable `testlib` helpers
