# CaveWhere 2026.4.3 Release Notes

**CaveWhere 2026.4.3** is a feature and bug-fix release. Headline additions: leads now render on exported maps with auto-placed labels and leader lines, clone errors get human-readable messages and provider-aware links, settings pages gain a Restore Defaults button, and the 3D view's background and grid lines now respect dark mode.

---

## New Features

### Capture and Export

- **Leads on exported maps** — uncompleted leads now render in PNG/JPG/TIFF/SVG/PDF exports as a "?" glyph plus the lead description (with size appended when both dimensions are set). A new "Leads" checkbox under each layer in Map Layers toggles them on a per-layer basis; default off so existing exports are unchanged (#468)
- **Auto-placed lead labels with leader lines** — lead descriptions are no longer pinned at a fixed offset that often landed on cave passages or overlapped each other. Labels are now placed in empty space using a distance-transform of the rendered tile alpha and connected to their "?" markers by leader lines routed below the captured tiles (so the leader shows through transparent regions and hides behind opaque ink). Placement scores candidates by direction (perpendicular to walls mid-passage, parallel at passage ends), avoids centerline legs, and rejects routes that would cross other leader lines. Preview now matches the 300 DPI export font size and position. Capture rendering uses the configured cwFontSettings family instead of `.AppleSystemUIFont`, fixing fonts that SVG viewers could not render

### Remote Projects

- **Friendlier clone error messages** — clone failures are now classified (Auth, NotFoundOrAccess, HostUnreachable, Other) and rendered with provider-aware text. A 404 on GitHub links to your pending invitations page instead of back to the 404ing repository URL. Hosting-provider knowledge (GitHub, GitLab, Bitbucket, generic) is consolidated into a single `cwGitHostingProvider` data table and reused by the deep-link handler (#490)
- **Show retry reason during GitHub device-auth polling** — the auth flow now displays the last poll outcome (e.g. `authorization_pending`, `slow_down`) beneath the countdown so users see what's happening between retries. Unknown poll errors no longer flip the UI into an Error state while the poller keeps running underneath (#472)

### Editing and Workflow

- **Restore Defaults on settings pages** — Jobs, Warping, PDF / SVG, and Appearance settings pages each get a "Restore Defaults" button that is disabled when values already match defaults. Also corrects the stale PDF help text that claimed the default was 300 ppi (the actual default is 96 ppi) (#148)
- **Copy All button on the Error Dialog** — copies every warning and fatal issue to the clipboard with `[Warning]` / `[Fatal]` prefixes, suitable for pasting into bug reports (#362)
- **Right-click "Show in Finder" on note and LiDAR thumbnails** — reveals the underlying image or `.glb` in the OS file manager. Hidden for bundled `.cw` archives and unsaved temporary projects (whose files live in a temp directory)
- **Source → Add → New now matches File → New** — the menu item no longer pops a name/parent-directory dialog; it just creates an empty project and navigates to the View page (#432)
- **Auto-select newly added note** — adding a note to a non-empty trip now moves the gallery selection to the new note instead of leaving the previous one selected (#238)
- **Warn when placing a station or lead outside a scrap** — clicking Add Station or Add Lead on a note with no scraps used to do nothing silently. An auto-hide error box now appears on the note image, and a help quote box points at the Add Scrap toolbar button while the user is in add mode with zero scraps (#182)

### Import / Export

- **Use Survex `*title` to name trips** — Survex `*begin` block names can't contain spaces but CaveWhere trip names can. Import now reads `*title` and prefers it over the block name; export emits `*title` so trip names with spaces round-trip (#183)
- **Tolerate unknown Survex `*data` columns** — unrecognized column keywords (e.g. `backtape`) used to abort the entire import with an error. They now emit a warning and are treated like `ignore`, mirroring Survex's own behavior. Adds `backbearing` / `backgradient` as aliases (#173)
- **Export Compass station names in uppercase** — Compass `.dat` files are case-sensitive while CaveWhere is not, so the old lowercase output produced names that did not match typical Compass conventions (#213)

### Theme

- **3D view background gradient and grid lines follow Theme** — the radial background gradient and grid line color were hardcoded in C++ and stayed light-mode in dark mode. Both now come from `Theme.qml` and switch with the existing dark flag (#497)

### Documentation

- **Privacy policy covering Git and GitHub integration** — documents OAuth scopes, GitHub REST endpoints, keychain-backed token storage, ssh-agent / `$HOME/.ssh` private-key lookup, and what flows off the device during Git and Git LFS operations

## Bug Fixes

### Sketching and Scraps

- **Fix profile scrap rotated by declination during auto-calc** — `updateNoteTransformation()` applied `northAdjustedForDeclination` to every scrap type. Only Plan needs that — Profile scraps are gravity-up, so the storage shift had no render-side back-out and the up direction rotated by exactly the declination (15.2° at Nimbus's 15° declination). The adjustment is now gated on Plan (#486)

### Capture and Export

- **Fix zero-sized capture bounds before tiles render** — centerline / leads / lead-leader children are now created after tile rendering finishes, so the QML capture item appeared at width=height=0 until the first tile arrived, breaking interaction. The viewport falls back to its expected size when `sceneBoundingRect()` is empty
- **Lead "?" glyph circle is now transparent** — drops the white fill so underlying passage ink shows through; only the outline and "?" remain

### Project Save and Load

- **Fix Save As `.cwproj` to cloud storage volumes** — `QDir::rename` cannot cross filesystem boundaries, so Save-As to a pcloud or similar mounted volume failed. A new robust move tries rename first and falls back to copy + recursive delete, with safety guards that reject empty / relative / root / `..`-containing / non-directory destinations, refuse to copy into a destination inside the source, and refuse to traverse symlinks. Source-cleanup failures after a successful copy are now surfaced as warnings instead of dropped (#475)
- **Fix legacy trip calibration load** — `cwRegionLoadTask::loadTripCalibration()` duplicated the proto deserialization and was missing the `has_frontsights()` / `has_backsights()` presence checks, so a missing field defaulted `frontSights` to `false` instead of `true`. Now delegates to the shared `cwProtoUtils::fromProtoTripCalibration()` helper (#387)
- **Clear commit fields after Commit All Changes** — the subject and description fields used to retain the previous commit's text after committing (#480)

### Survex / Walls / Compass

- **Fix cross-thread `setParent` in tree-import** — `cwSurvexImporter` and `cwWallsImporter` built the node tree on a worker thread and then tried to reparent under a GlobalData object that lived on the main thread. Qt rejected the cross-thread reparenting with a warning and silently dropped the assignment, so `caves()` / `trips()` came out empty and the importer appeared to "not import any data." Each top-level node tree is now moved onto its own thread before reparenting
- **Fix #481: open Walls / Survex import dialog above the main window** — without a transient parent, the dialog could appear behind the main window on macOS after the importer finished
- **Fix #435: skip empty shots so cavern can run on the rest of the cave** — empty shot rows were exported as `from to - - -` stub lines that cavern rejected with "Tape reading may not be omitted", killing the line plot for the entire cave. The geometry walker also bootstrapped from station index 0, so any chunk with an empty leading row had its entire geometry skipped. Empty shots and empty LRUD rows are now skipped on export, and the line-plot walker scans forward to the first known station

### UI and Theme

- **Fix #342: exit carpet tool when clicking away** — activating a carpeting sub-tool (Add Scrap, Add Station, Add Lead, Select) and then navigating to a different page or switching to a different note left NoteItem in the tool state. The gallery now resets on `currentNote` and page-address changes
- **Fix #477: readable TextField placeholder color in dark mode** — Fusion (Windows / Linux) and Material (Android) ship an essentially unreadable placeholder color on dark surfaces, so welcome-page hints disappeared in dark mode. Overrides `palette.placeholderText` to `Theme.textSubtle`
- **Fix #479: fade in Label3d on creation when initially visible** — the fade-in only ran on `onVisibleChanged`, so the compass N/S/E/W labels in `Compass.qml` (visible from creation) stayed at opacity 0.0
- **Fix oversized, pixelated lock buttons in camera options** — the azimuth / pitch LockButton used 64×64 PNG icons that Qt scaled up on high-DPI displays. Switched to twbs-icons SVGs pinned at 16 px (#363)

### Platform and Build

- **Fix MSVC C4715 warning in `GitMergeFutureWatcher`** (#465)
- **Gate AddressSanitizer on macOS Debug behind `ASAN_ENABLED`** — previously ASAN was unconditionally enabled for Darwin Debug builds, blocking profiling or heap-sensitive local runs without editing the CMakeLists. The option defaults ON so existing behavior is preserved
