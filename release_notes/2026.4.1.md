# CaveWhere 2026.4.1 Release Notes

**CaveWhere 2026.4.1** is a patch release focused on fixing legacy project conversion issues and adding a small export feature.

---

## New Feature

- **Scale bar visibility toggle** — each map export layer now has a checkbox to show or hide the scale bar, giving you cleaner exports when the scale bar isn't needed (#416)

---

## Bug Fixes

### Legacy Project Conversion (v6)

Several issues were fixed when converting older v6 project files to the new format:

- **Fix EXIF rotation handling** — portrait-oriented note images that were auto-rotated by EXIF metadata now convert with correct scrap coordinates, fixing distorted scraps on affected notes
- **Fix note ordering** — trips with 10 or more notes no longer have their order scrambled during conversion (notes were sorted lexicographically as "1, 10, 11, 2, 3..." instead of numerically) (#420)
- **Fix image resolution loss** — custom DPI values (e.g. 800 DPI) are now preserved during conversion instead of being silently reset to the image file's embedded DPI (e.g. 72)
