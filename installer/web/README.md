# cavewhere.com Web Files

These three files form the WordPress **child theme** (`boldgrid-resolve-child`) hosted on cavewhere.com. They are tracked here so that changes are versioned alongside the CaveWhere app code they depend on.

## Files

| File | Server path | Purpose |
|------|-------------|---------|
| `style.css` | `wp-content/themes/boldgrid-resolve-child/style.css` | Child theme declaration — imports parent theme styles |
| `functions.php` | `wp-content/themes/boldgrid-resolve-child/functions.php` | Enqueues parent and child stylesheets |
| `page-open.php` | `wp-content/themes/boldgrid-resolve-child/page-open.php` | Custom page template for `cavewhere.com/open` — handles deep-link handoff |

## How `page-open.php` works

`https://cavewhere.com/open?repo=<github_url>` is the share link generated by CaveWhere's **File → Share** feature. When a recipient visits this URL:

1. The page redirects their browser to `cavewhere://open?repo=<github_url>`
2. macOS/Windows intercepts the `cavewhere://` scheme and opens the app
3. If the app doesn't open within 3 seconds the page shows a Download CaveWhere button
4. When the user switches back to the browser after the app opens, the heading changes to "Map Cave"

The page validates the `repo` param client-side — only `https://` URLs on `github.com`, `gitlab.com`, or `bitbucket.org` are accepted. Invalid or missing params show an error state instead of redirecting.

## Deploying changes

Upload the modified file(s) via SFTP or the WordPress admin file editor to the paths listed in the table above. No build step is needed — WordPress picks up template changes immediately.

## Keeping the allowlist in sync

`page-open.php` enforces a host allowlist (`github.com`, `gitlab.com`, `bitbucket.org`) independently of the app. If you add or remove a host in `cavewherelib/src/cwDeepLinkHandler.cpp`, update the same list in the `isValidRepo()` function in `page-open.php`.

## WordPress page setup

The `cavewhere.com/open` page must have the **"CaveWhere Open"** page template selected in WordPress (Pages → Open → Page Attributes → Template). This is already configured on the live site — only needs to be set again if the page is recreated from scratch.
