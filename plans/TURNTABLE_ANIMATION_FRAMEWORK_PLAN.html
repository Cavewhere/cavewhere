Turntable Animation Framework

Extend cwBaseTurnTableInteraction with a state-based animator. Replace the matrix-element-wise lerp used by centerOn today with a 5-channel state interpolator that handles rotation and zoom correctly, so the sink-clip preview (and future features) can animate the full camera, not just translation.

1. Why the current animation is not enough

Today there is exactly one camera animation primitive in the turntable:

cwBaseTurnTableInteraction::ViewMatrixAnimation  // cwMatrix4x4Animation, duration 1000 ms, InOutCubic

It is consumed by centerOn(point, animate=true) (cwBaseTurnTableInteraction.cpp:59-81) and by the only external caller CaveLeadPage.qml:112. The class cwMatrix4x4Animation interpolates a QMatrix4x4 element-by-element:

// cwMatrix4x4Animation.cpp:41-45
for (int i = 0; i < 16; i++) {
    result.data()[i] = fromValue + (toValue - fromValue) * progress;
}

That is a numeric lerp, not a rigid-body interpolation. It works for centerOn only because centerOn animates pure translation — the rotation block of the matrix is identical at from and to, so each row stays on its own straight line. The moment you try to animate a rotation, the intermediate matrices are skewed/scaled non-orthogonal blends:

Worse, the turntable's stored rotation state (Pitch, Azimuth, CurrentRotation) is never touched during an animation. After a successful translation animation the stored state still matches the view matrix, so the next updateRotationMatrix() (cwBaseTurnTableInteraction.cpp:417-433) computes rotationDifferance = identity and everything is consistent. After a (hypothetical) rotation animation, the stored state would not match the animated matrix — the first user rotate drag would snap the camera back to the stored orientation.

Either of those two problems is fatal for the sink-clip preview feature. Both together mean we cannot just throw more cwMatrix4x4Animations at the wall.

2. What the camera state actually is

The turntable's view matrix is reproducible from a small set of independent channels:

ChannelTypeStored asNotes
centerQVector3Dprivate member (today: LastMouseGlobalPosition)World-space orbit pivot. Used directly by updateRotationMatrix at lines 428-430.
azimuthdoubleAzimuth member, exposed via Q_PROPERTYDegrees, wrapped to [0, 360) by clampAzimuth.
pitchdoublePitch member, exposed via Q_PROPERTYDegrees, clamped to [-90, 90] by clampPitch.
distancedoublederivable from Camera->viewMatrix() + centerCamera-to-center world distance. Encodes perspective zoom.
zoomScaledoubleCamera->zoomScale()Ortho only. Ignored for perspective projection.

Given these five channels and the camera's projection mode, the view matrix is determined. Conversely, the current view matrix can be decomposed back into them (with center read directly from the member field rather than re-derived). This is the basis for the animation framework: interpolate channels, rebuild view matrix.

3. cwTurnTableViewState — the value type

New header cavewherelib/src/cwTurnTableViewState.h. A Q_GADGET (no QObject) returned and passed by value, registered as a metatype so it round-trips through QML, signals, and animations.

class cwTurnTableViewState
{
    Q_GADGET
    QML_VALUE_TYPE(turnTableViewState)

    Q_PROPERTY(QVector3D center MEMBER center)
    Q_PROPERTY(double azimuth         MEMBER azimuth)
    Q_PROPERTY(double pitch           MEMBER pitch)
    Q_PROPERTY(double distance        MEMBER distance)
    Q_PROPERTY(double zoomScale       MEMBER zoomScale)

public:
    QVector3D center;
    double    azimuth   = 0.0;
    double    pitch     = 90.0;
    double    distance  = 50.0;
    double    zoomScale = 1.0;

    bool operator==(const cwTurnTableViewState&) const = default;
};

Q_DECLARE_METATYPE(cwTurnTableViewState)

Default values mirror resetView() (cwBaseTurnTableInteraction.cpp:238-258): pitch 90, azimuth 0, view-matrix translation (0, 0, -50), ortho zoom at default scale.

The same struct can be returned by other model-side helpers in lieu of QVariantMap — the user's earlier previewBoundsFor(int) on cwSinkTrainingModel will return its own Q_GADGET (cwSinkPreviewBounds) in a sibling plan; this one is scoped to the turntable.

4. Renaming LastMouseGlobalPosition

The field is set in startPanning (line 163) and startRotating (line 206) and consumed by updateRotationMatrix (line 428-430) as the world-space pivot point of the orbital rotation. It is not a mouse position; it is the orbit center expressed as a world-space point.

BeforeAfter
QVector3D LastMouseGlobalPosition;QVector3D m_center;
LastMouseGlobalPosition = unProject(mappedPosition);m_center = unProject(mappedPosition);

The other rotation-state members (Pitch, Azimuth, CurrentRotation, LastMousePosition, ViewMatrixAnimation) keep their current names — they read fine, and renaming them is not the point of this plan.

5. The center lock

Two new public methods replace the "external pivot override" idea from earlier discussion:

Q_INVOKABLE void setCenter(QVector3D center);   // writes m_center, emits notify
QVector3D center() const { return m_center; }

Q_INVOKABLE void setCenterLocked(bool locked);
bool isCenterLocked() const { return m_centerLocked; }

When locked, both startRotating and startPanning skip the unProject(position) assignment — the externally-set center stays put across drags. When unlocked (default), behavior matches today bit-for-bit.

void cwBaseTurnTableInteraction::startRotating(QPoint position) {
    position = Camera->mapToGLViewport(position);
    if (!m_centerLocked) {
        m_center = unProject(position);
    }
    LastMousePosition = position;
}

The lock is the primitive used by the sink-clip preview to keep rotation orbiting the previewed sink. It is independent of the animation framework — locking does not require an animation, and animations can run without the lock engaged.

6.3 Per-tick application

setViewState(s) is the single point that writes all five channels back into the live turntable + camera. Pseudo-code:

void cwBaseTurnTableInteraction::setViewState(const cwTurnTableViewState& s) {
    if (!Camera) return;

    Pitch          = clampPitch(s.pitch);
    Azimuth        = clampAzimuth(s.azimuth);
    m_center = s.center;

    // Rebuild view matrix from (pitch, azimuth, distance, center)
    QQuaternion pitchQuat   = QQuaternion::fromAxisAndAngle(1,0,0, Pitch);
    QQuaternion azimuthQuat = QQuaternion::fromAxisAndAngle(0,0,1, Azimuth);
    QQuaternion newQuat     = pitchQuat * azimuthQuat;
    setCurrentRotation(newQuat);

    QMatrix4x4 view;
    view.translate(0.0, 0.0, -s.distance);
    view.rotate(newQuat);
    view.translate(-s.center);
    Camera->setViewMatrix(view);

    if (Camera->projection().type() == cwProjection::Ortho) {
        Camera->setZoomScale(s.zoomScale);
    }

    emit pitchChanged();
    emit azimuthChanged();
    emit centerChanged();
}

viewState() is the inverse: read Pitch, Azimuth, m_center, compute distance as |cameraWorldPos − m_center| where cameraWorldPos falls out of inverting Camera->viewMatrix(), and read Camera->zoomScale().

6.4 Animation tick

void cwBaseTurnTableInteraction::animateToViewState(
        const cwTurnTableViewState& target, int durationMs) {
    stopAnimation();                  // also stops ViewMatrixAnimation
    m_stateAnimStart  = viewState();
    m_stateAnimTarget = target;
    m_stateAnimation->setDuration(durationMs);
    m_stateAnimation->setStartValue(0.0);
    m_stateAnimation->setEndValue(1.0);
    m_stateAnimation->start();
}

// connected to m_stateAnimation::valueChanged
void cwBaseTurnTableInteraction::onStateAnimTick(const QVariant& v) {
    const qreal t = v.toReal();
    cwTurnTableViewState s;
    s.center = lerp(m_stateAnimStart.center,
                          m_stateAnimTarget.center, t);
    s.azimuth      = lerpAzimuth(m_stateAnimStart.azimuth,
                                 m_stateAnimTarget.azimuth, t);
    s.pitch        = lerp(m_stateAnimStart.pitch,    m_stateAnimTarget.pitch,    t);
    s.distance     = isPerspective
                     ? lerp(m_stateAnimStart.distance, m_stateAnimTarget.distance, t)
                     : m_stateAnimTarget.distance;   // ortho: pin to target, don't animate
    s.zoomScale    = lerp(m_stateAnimStart.zoomScale, m_stateAnimTarget.zoomScale, t);
    setViewState(s);
}

lerpAzimuth wraps around the shorter arc on a 0..360 circle — going from 350° to 10° should be a 20° forward sweep, not a 340° backward sweep. lerp for pitch is straight linear (range is bounded ±90 and never wraps). Distance is lerped only for perspective projection — in ortho the visible scale comes from zoomScale, so animating distance is pointless flicker on the depth buffer. zoomScale is straight linear; a log-scale option for distance can be a parameter on the state struct later if zooming feels non-linear.

Distance correctness across projection switches. When the user toggles between ortho and perspective (handled separately by cwMatrix4x4Animation, outside this framework), the next call to viewState() recomputes distance from the current view matrix: cameraWorldPos = inverse(viewMatrix) · (0,0,0), distance = |cameraWorldPos − m_center|. So the captured state always carries a correct distance for the projection mode the camera is currently in, even after a mode swap.
Why not SLERP a quaternion? Because the turntable's two angles are orthogonal (pitch around X, azimuth around Z, applied in that fixed order). Lerping each scalar independently produces the same path as a SLERP composed of those two axes, with simpler code and no risk of antipodal-quaternion sign flips. The composition order is fixed by updateRotationMatrix (cwBaseTurnTableInteraction.cpp:421-423).

7. centerOn rewritten on the framework

The new body is short and obviously equivalent for translation-only animations:

void cwBaseTurnTableInteraction::centerOn(QVector3D point, bool animate) {
    cwTurnTableViewState target = viewState();
    target.center = point;
    // distance, azimuth, pitch, zoomScale preserved
    if (animate) {
        animateToViewState(target);
    } else {
        setViewState(target);
    }
}

Compared to the existing implementation (cwBaseTurnTableInteraction.cpp:59-81), this preserves the observable contract: the point ends up at screen center, orientation and zoom are unchanged. The reconstruction in setViewState places the camera at exactly the right world position because distance was captured from the current view and the rotation block is rebuilt from the unchanged (pitch, azimuth).

Verify with a regression test, not by inspection. The existing centerOn derives the translation from screen-center unprojection at the source point's depth — a slightly different code path. A test with a known camera + a known point should confirm that both implementations land at the same final view matrix to within float epsilon.

8. zoomTo is left alone (for now)

zoomTo(QBox3D) (cwBaseTurnTableInteraction.cpp:633-728) currently calls resetView(), then computes a fit, then snaps the camera. It does not animate. Folding it into the animator is a separate plan because:

  • The fit math (ortho viewport-span vs. perspective FOV) is non-trivial and is the bulk of the function.
  • The current "resets to default orientation" behavior is depended on (or is at least a known quantity) at every call site.
  • The sink-clip preview feature does not need zoomTo animation — it builds its target view state explicitly and calls animateToViewState directly.

A follow-up plan can refactor zoomTo to compute a target state and dispatch through the animator.

9. Commit plan

Three commits on the same branch. Each commit must build green and pass all tests. The test file is started in commit 1 and grown commit-by-commit.

Commit 1 — Regression tests against current code

No changes to cwBaseTurnTableInteraction in this commit. The point is to pin existing behavior with executable assertions so that the refactors in commits 2 and 3 cannot silently regress it. Tests must pass on the unmodified base class before any rename or new method is introduced.

New file: testcases/test_cwBaseTurnTableInteraction.cpp. Catch2 cases under tag [cwBaseTurnTableInteraction]:

  • defaultViewAfterReset: a freshly constructed interaction with a connected camera has azimuth() == 0, pitch() == 90, view matrix translates (0, 0, -50), ortho zoomScale at camera default.
  • setAzimuth/setPitch update view matrix: setting each rotates the camera view by the expected quaternion delta around the current LastMouseGlobalPosition.
  • azimuthLocked/pitchLocked block setters: with the lock on, calling setAzimuth/setPitch is a no-op (matches the current guard in setAzimuth/setPitch at .cpp:545,561).
  • centerOn(point, false) translation: for a known camera + point, the resulting view matrix translates the point to screen center. This baseline lets commit 3 prove parity after the refactor.
  • centerOn(point, true) end-state: drive the animation to completion via QTest::qWait or by stepping the animation, assert the final view matrix matches the false case.
  • zoomTo(box) end-state: after zoomTo, orientation is at defaults and the box is framed within the viewport. Pins the existing behavior so commit 3 doesn't accidentally break it.
  • startRotating updates LastMouseGlobalPosition: after startRotating(p), the field equals unProject(p). This baseline lets commit 2 prove that the center-lock change only gates this assignment.

Commit 2 — Rename, state value type, lock, snap setters

Modifies cwBaseTurnTableInteraction structurally but does not yet add animation. After this commit, centerOn still uses the old cwMatrix4x4Animation path.

  • Rename LastMouseGlobalPosition → m_center throughout the .cpp.
  • Migrate azimuthLocked, pitchLocked, gridPlane from Q_OBJECT_BINDABLE_PROPERTY_WITH_ARGS to plain members + setter + Q_PROPERTY + NOTIFY. Drop BINDABLE from the Q_PROPERTY lines; drop azimuthLockedBinding(), pitchLockedBinding(), bindableGridPlane() (no external callers — verify with grep before deleting).
  • Add new header cavewherelib/src/cwTurnTableViewState.h declaring the Q_GADGET.
  • Add center() / setCenter(), isCenterLocked() / setCenterLocked(), plus Q_PROPERTY declarations and NOTIFY signals.
  • Gate the unProject assignment in startRotating and startPanning on m_centerLocked.
  • Add viewState() reading current state (including projection-aware distance via inverse-view-matrix camera position) and setViewState() applying it.

New test cases in test_cwBaseTurnTableInteraction.cpp:

  • State round-trip (ortho and perspective): build a known state, setViewState, read back via viewState(), compare to float epsilon.
  • Center lock blocks startRotating: setCenter(c); setCenterLocked(true); startRotating(somePoint); → center() == c.
  • Center lock blocks startPanning: same shape.
  • Center lock off restores old behavior: with the lock off, the existing commit 1 regression test for startRotating still passes (gives confidence the gate is purely additive).
  • Bindable migration: setting azimuthLocked/pitchLocked still blocks rotation; the NOTIFY signal still fires.

Commit 3 — Animation framework

  • Add private member QVariantAnimation* m_stateAnimation, two snapshot members for start/target state, animationFinished signal.
  • Implement animateToViewState(target, durationMs) and the per-tick handler. Distance is lerped only for perspective.
  • Rewrite centerOn(point, true) on top of animateToViewState. The animate=false branch becomes setViewState(target).
  • cwMatrix4x4Animation and ViewMatrixAnimation remain in place; they are still used for projection-mode swap animations.

New test cases:

  • centerOn parity: the commit 1 baseline for centerOn(point, false) view matrix is reproduced exactly by the refactored implementation.
  • Animator end-state: animateToViewState(target, 100ms), run to completion, assert viewState() == target.
  • Animator mid-tick: at t=0.5, pitch / azimuth / center are each at the midpoint, zoomScale is at the midpoint, distance is at the midpoint in perspective and pinned to target in ortho.
  • Azimuth short-arc wraparound: 350° → 10° at t=0.5 is 0°, not 180°.
  • CaveLeadPage smoke: in the QML test suite, the existing lead-goto-zoom flow still lands at the correct world point — pins that the centerOn public contract is preserved.
