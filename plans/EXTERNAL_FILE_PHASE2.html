Cross-cutting test rules (per-process safety, cavern serialisation, narrow tag filters, "existing tests always green") live in master §10.1 and apply to every commit here without restating.

Phase 2 goal: a user can attach an external centerline to a trip through the UI, see the resolved stations, attach a note to one, and watch an edit in the project tree trigger a re-solve. The CavernOutputPage grows the attached-centerlines list and driver-source disclosure.

Direction change (2026-07-16): live-link auto-reconcile dropped — notify + one-click update

Decided after commit 4. The watcher-driven auto-reconcile model (source file changes on disk → CaveWhere silently copies it into the project and re-solves) is dropped. The complexity it generated was out of proportion to its value: mid-session refreshes with nobody at the wheel forced the commit-4 dirty-bit semantics, the mid-git-sync fold-in rules, the per-machine last-sync-wins ping-pong corollary, and made commit 8's async scan pipeline load-bearing — and the UX was confusing (plot changes and save prompts the user never caused). (Commit 8 was later re-promoted for a different reason: the commit-7 recompute added a synchronous main-thread staleness probe and row rebuild — see the commit-8 entry.)

The replacement model:

  • The in-project copy is the source of truth. It is what solves, syncs, and versions. The user's original file is just where updates come from.
  • The source path is always remembered in cwExternalSourceSettings (per-machine, as built in commit 2). The attach dialog's live-link checkbox is dropped; there is no mode dichotomy. A "Forget source" action clears the entry for users who don't want update checks. isLiveLink() reads as "has a remembered source" (rename to hasSource() when next touching the class).
  • Staleness is detected, never acted on automatically. The source-side QFileSystemWatcher stays but its only action is setting a per-owner stale flag (staleSourceOwners, mirroring missingSourceOwners); an open-time probe backstops it (the freshness test reuses computePlan and is side-effect-free; its I/O cost is bounded by the closure size and moves off the main thread in commit 8). Watcher failure modes become benign — worst case the banner lights late.
  • Updating is user-triggered. The trip panel shows "The source file has changed on disk — [Update]"; Update runs the existing scan + reconcile + re-solve path. The commit-4 dirty-bit wiring is unchanged and now always fires as the direct result of a click. Updates cannot interleave with a git sync (the affordance is disabled while a sync runs), so the §10 mid-sync row is moot.
  • Missing source is benign: the Update affordance disables with a "source not found" hint; no urgent banner.

Reading guide for the rest of this file and the master plan: text written before this note says "live-link" for what is now simply "a remembered source"; "reconcile-from-source on watcher event" is now "flag stale on watcher event; reconcile on Update". Normative sections below (§6, §10, §11 commits 5/7/8, §12 gates) have been updated; incidental mentions elsewhere have not. The auto-reconcile stripping from cwLinePlotManager::onWatchedFileChanged lands in commit 7 (the manager commit).

6. Attach orchestrator design

The orchestrator composes the three primitives Phase 1 shipped (scanner, reconcile, data-model mutation) plus the new cwExternalSourceSettings source-memory entry write into one cancellable, testable entry point. Free functions in cwExternalCenterlineAttach:: namespace; no QObject, no member state. 

// cwExternalCenterlineAttach.h
namespace cwExternalCenterlineAttach {

// AttachOptions was dropped at commit 5: after the direction change its
// only field (sourcePath) duplicated the sourceFile parameter - the path
// the user picked is both the scan target and the remembered source.

// SeededTripMetadata lives on the scanner (see section 7):
// cwExternalCenterlineScanner::SeededTripMetadata - the orchestrator
// reuses it rather than defining its own copy.

struct AttachReport {
    cwExternalCenterlineScanner::ScanResult scan;  // copy of scanner result
    cwExternalCenterline persisted;                 // entryFile that landed on the trip
    QStringList warnings;                           // scanner warnings + reconcile warnings
    cwExternalCenterlineScanner::SeededTripMetadata metadata;  // fields actually applied
                                                    // (skipped fields stay empty)
};

// Trip-level attach: scan sourceFile -> set externalCenterline ->
// reconcile into trip's attachment dir -> seed trip metadata ->
// remember sourceFile in cwExternalSourceSettings (always).
QFuture<Monad::Result<AttachReport>>
attach(cwTrip* trip,
       const QString& sourceFile,
       cwSaveLoad* saveLoad,
       cwExternalSourceSettings* externalSourceSettings);

// Trip-level detach: clear externalCenterline -> remove attachment dir
// -> drop any cwExternalSourceSettings entry. Idempotent; safe to call on a
// trip that's already Native.
QFuture<Monad::ResultBase>
detach(cwTrip* trip,
       cwSaveLoad* saveLoad,
       cwExternalSourceSettings* externalSourceSettings);

} // namespace

Sequencing details:

  • Scan first, persist last. attach() runs the scanner via QtConcurrent::run so the dialog can show progress and cancel. Only on Monad::Ok does the function call cwTrip::setExternalCenterline and reconcile. A cancelled or failed scan leaves the data model untouched. Decided 2026-07-19 (set-model-on-success), now landed: the model flip sits after the reconcile verify passes — originally setExternalCenterline fired at scan-success, before any file landed, so observers saw a file-less attachment for the whole copy drain (a crash in that window persisted a broken attachment via the externalCenterlineChanged save hook). Reconciling first (the attachment dir comes from externalCenterlineDir(trip), which doesn't need the model) makes failure mean "nothing happened" and deleted the rollback path, including its dangling-previous fallback. Landed as the small commit before commit 9 (see the commit sequence) so the wrapper and dialog bind to the final semantics.
  • Reconcile target. saveLoad->externalCenterlineDir(trip) resolves the attachment dir; the orchestrator passes it to cwExternalCenterlineSync::reconcile directly.
  • Source-memory wiring (rewritten by the direction change). The orchestrator always writes a cwExternalSourceSettings::ExternalCenterlineSource entry with ownerId = trip->id() and sourcePath = sourceFile — the path the user picked is always known at attach time, and there is no mode toggle. AttachOptions is dropped entirely. Detach (and a later "Forget source" action) removes the entry. The cwExternalSourceSettings store is the same rootData-owned object cwLinePlotManager observes; the manager recomputes its watch set (stale/missing probing only, no auto-reconcile) on externalCenterlineSourcesChanged, so the orchestrator just writes.
  • Metadata seeding. Scanner returns SeededTripMetadata alongside dependencies. The orchestrator applies it to cwTripCalibration only when each field is set and the trip's existing value is empty (so re-attach over user-edited metadata never clobbers). Per §8.8 question 4. Declination is special (master §8.8 q7): fileOwnsDeclination() means the file owns it — the seeded value is display-only (panel renders it read-only, or "automatic (IGRF)" for declinationIsAuto, with autoDeclination = false) and the driver emits no declination for the trip; otherwise the trip stays CaveWhere-owned (editable, IGRF auto default, injected into the driver).
  • Detach on a Native trip is a no-op. Idempotent so the QML caller doesn't need to guard.
  • Persistence wiring (found missing at commit 5). cwSaveLoad's connectTrip/connectCave blocks had no connection for externalCenterlineChanged, so a changed attachment would never re-save the owner's protobuf (and never flip the modified bit through saveObject). Commit 5 added both connections; the attach round-trip test guards the trip one. The cave connection has no coverage until Phase 3's cave-level attach — its round-trip test must guard it then.
  • QML entry point. A thin Q_INVOKABLE QFuture attachTripCenterline(cwTrip*, QString sourcePath) wrapper on cwRootData exposes the orchestrator to QML, resolving the cwSaveLoad and cwExternalSourceSettings dependencies from the root singleton and forwarding. Decided at commit 5: cwRootData, not a new singleton — it already owns every dependency and a one-function manager is bloat. The wrapper itself lands with the first QML consumer (commit 9's attach dialog). Decided at the commit-7 review: when the wrapper lands, per-owner mutual exclusion is enforced in one place instead of by per-affordance UI bindings — generalize the manager's m_updateInFlightOwners into a per-owner operation token covering attach, detach, and updateFromSource, exposed as a single Q_INVOKABLE isOwnerBusy(ownerId) (+ change signal). The wrapper refuses attach/detach for a busy owner, so non-UI callers get the same protection, and QML binds one query for the Update/Detach/re-attach affordances instead of N bespoke bindings. cwScopeStationListModel backs both the panel's stations list and the carpet sub-page picker autocomplete.         QualifiedNameRole,                   // "qualifiedName": full qualified name

    };

};

Rules:

  • Prefix strip. A station's qualified name cave_x.trip_y.a1 with scopePrefix = "cave_x.trip_y." displays as a1; cave_x.trip_y.sidepassage.b1 displays as sidepassage.b1. Per §8.8 question 9. Both roles are canonical lowercase — cwSurveyNetwork canonicalizes keys via cwStation::canonicalKey (toLower()), so input case is unrecoverable at this layer; the prefix is canonicalized the same way, so matching is case-insensitive. Carrying original-case display names through the solve is a separate product decision (blast radius: cwSurveyNetwork, cwLinePlotTask, cwSurvey2DGeometryRule, this model). Decided 2026-07-19: v1 ships canonical lowercase display names; carrying original case through the pipeline is additive API and is parked in the §7 deferred-refactor list — it does not gate commits 9+.
  • Sort. Stable lexicographic sort by qualified name. Locked here so the carpet picker autocomplete is deterministic regardless of how cavern ordered its output. (Keys are canonical lowercase, so case-insensitivity is inherent, not a comparator property.)
  • Refresh. The network value property is bound (in QML) to cwLinePlotManager::regionNetwork; the setter walks the new network's stations() / position() once, synchronously, and re-populates the row list. Setting an equal network is a no-op (no reset, no signal). No incremental diff; project sizes don't warrant it in v1. All the async lives in the manager — the model itself has no futures.
  • Empty prefix. scopePrefix.isEmpty() returns zero rows (rather than every station in the region). The caller is expected to set the prefix before the model has any meaning.
  • Empty network. Setting a default-constructed cwSurveyNetwork clears the rows synchronously. Errored solves never publish, so regionNetwork (and therefore the model) keeps the last good network until the next successful solve.

B1 — Attach produces no centerline in the 3D renderer (FIXED 2026-07-21)
