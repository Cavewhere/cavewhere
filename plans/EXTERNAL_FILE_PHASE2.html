External File Integration — Phase 2: Trip-level attach + CavernOutputPage v1

Part of the External File Integration plan. Read the master plan first — it owns architecture (§2), data model (§3), scanner API shape (§4), copy workflow (§5), driver shape (§6), consumer-side pipeline (§7), UX (§8), risks (§11), and deferred items (§12). 6. Attach orchestrator design

The orchestrator composes the three primitives Phase 1 shipped (scanner, reconcile, data-model mutation) plus the new cwLocalSettings live-link entry write into one cancellable, testable entry point. Free functions in cwExternalCenterlineAttach:: namespace; no QObject, no member state. 

// cwExternalCenterlineAttach.h
namespace cwExternalCenterlineAttach {

struct AttachOptions {
    bool liveLink = false;       // false = import (copy + forget source path)
    QString sourcePath;          // absolute path; ignored when liveLink is false
};

struct SeededTripMetadata {
    std::optional<QDate> date;
    std::optional<double> declination;  // degrees, positive east
    QString team;
};

struct AttachReport {
    cwExternalCenterlineScanner::ScanResult scan;  // copy of scanner result
    cwExternalCenterline persisted;                 // entryFile that landed on the trip
    QStringList warnings;                           // scanner warnings + reconcile warnings
    SeededTripMetadata metadata;                    // applied to cwTripCalibration on success
};

// Trip-level attach: scan source -> reconcile into trip's attachment dir
// -> set externalCenterline -> seed trip metadata -> optionally update
// cwLocalSettings live-link entry.
QFuture<Monad::Result<AttachReport>>
attach(cwTrip* trip,
       const QString& sourceFile,
       AttachOptions options,
       cwSaveLoad* saveLoad,
       cwLocalSettings* localSettings);

// Trip-level detach: clear externalCenterline -> remove attachment dir
// -> drop any cwLocalSettings entry. Idempotent; safe to call on a
// trip that's already Native.
QFuture<Monad::ResultBase>
detach(cwTrip* trip,
       cwSaveLoad* saveLoad,
       cwLocalSettings* localSettings);

} // namespace

Sequencing details:

  • Scan first, persist last. attach() runs the scanner via QtConcurrent::run so the dialog can show progress and cancel. Only on Monad::Ok does the function call cwTrip::setExternalCenterline and reconcile. A cancelled or failed scan leaves the data model untouched.
  • Reconcile target. saveLoad->externalCenterlineDir(trip) resolves the attachment dir; the orchestrator passes it to cwExternalCenterlineSync::reconcile directly.
  • Live-link wiring. When options.liveLink is true, the orchestrator writes a cwLocalSettings::ExternalCenterlineSource entry with ownerId = trip->id() and sourcePath = options.sourcePath. When false, any existing live-link entry for that owner is removed (import mode is the default). The cwLocalSettings object is the same one cwLinePlotManager reads; the manager picks up the change via the setLocalSettings setter the orchestrator calls after the write.
  • Metadata seeding. Scanner returns SeededTripMetadata alongside dependencies. The orchestrator applies it to cwTripCalibration only when each field is set and the trip's existing value is empty (so re-attach over user-edited metadata never clobbers). Per §8.8 question 4.
  • Detach on a Native trip is a no-op. Idempotent so the QML caller doesn't need to guard.
  • QML entry point. A thin Q_INVOKABLE QFuture attachTripCenterline(cwTrip*, QString, bool liveLink, QString sourcePath) wrapper on cwRootData (or on a new cwExternalCenterlineManager singleton — decide during commit 2 based on cwRootData bloat) exposes the orchestrator to QML. The wrapper resolves the cwSaveLoad and cwLocalSettings dependencies from the root singleton and forwards.

7. Scanner SeededTripMetadata

Phase 1's ScanResult carries { canonicalEntry, dependencies, warnings }. Phase 2 adds seededMetadata to populate the trip's date / declination / team on first attach. The scanner pulls these from the entry file only — nested *included files are not consulted because their directives would scope to a child block, not the trip the user is attaching.

FieldSurvex sourceCompass sourceWalls source
date *date YYYY.MM.DD or *date YYYY-MM-DD SURVEY DATE: M D YYYY in the .dat header #DATE YYYY-MM-DD
declination *calibrate declination <degrees> CORRECTIONS: <decl> <tape> <clino> #UNITS DECL=<degrees>
team *team <name> (multiple lines joined with ", ") SURVEY TEAM: line #NOTE entries containing "team" (best-effort, may be empty)

Missing or unparseable values leave the corresponding std::optional empty and the team string empty. Per §8.8 question 4, this is best-effort seeding — the source of truth after first attach is the CaveWhere trip record.

8. Per-scope station-list model

cwScopeStationListModel backs both the panel's stations list and the carpet sub-page picker autocomplete. Read-only, list-shaped, one row per station whose qualified name in cwSurveyNetworkArtifact starts with the model's scopePrefix (typically cave_<X>.trip_<Y>.). 

// cwScopeStationListModel.h
class cwScopeStationListModel : public QAbstractListModel {
    Q_OBJECT
    QML_ELEMENT

    // The qualified-name prefix to filter on. Setter recomputes the
    // visible rows + emits modelReset (cheap: typical scope has <
    // a few hundred stations).
    Q_PROPERTY(QString scopePrefix READ scopePrefix WRITE setScopePrefix NOTIFY scopePrefixChanged)

    // The source of truth. Setter connects to the artifact's changed
    // signal so the model auto-refreshes when the line-plot pipeline
    // publishes a new network.
    Q_PROPERTY(cwSurveyNetworkArtifact* artifact READ artifact WRITE setArtifact NOTIFY artifactChanged)

public:
    enum Roles {
        StationNameRole = Qt::UserRole + 1,  // prefix-stripped display name
        QualifiedNameRole,                   // full qualified name
        PositionRole,                        // QVector3D from the lookup
    };
    ...
};

Rules:

  • Prefix strip. A station's qualified name cave_X.trip_Y.A1 with scopePrefix = "cave_X.trip_Y." displays as A1; cave_X.trip_Y.sidepassage.B1 displays as sidepassage.B1. Per §8.8 question 9.
  • Sort. Stable sort by qualified name case-insensitively. Locked here so the carpet picker autocomplete is deterministic regardless of how cavern ordered its output.
  • Refresh. Connected to artifact->changed() via AsyncFuture::observe(...).context(this, ...); on artifact reset, walks the new network's cwStationPositionLookup once and re-populates the row list. No incremental diff; project sizes don't warrant it in v1.
  • Empty prefix. scopePrefix.isEmpty() returns zero rows (rather than every station in the region). The caller is expected to set the prefix before the model has any meaning.
