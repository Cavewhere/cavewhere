Fixes #575 (hidden shots still pickable) and #549 (reset frames hidden geometry) through a single mechanism.

0 · Goal & the two bugs

Visibility in CaveWhere is keyed on keywords: the user filters by keyword (Cave, Trip, Year, Type…), a filter pipeline splits keyword items into an accepted (visible) and a rejected (hidden) set, and each item hides its geometry at whatever granularity it owns. That hiding is applied on the render side only. The intersecter — which answers picks and bounding-box queries — never learns about it, except at whole render-object granularity.

BugSymptomCause
#575 The Distance tool snaps to a hidden shot; turntable pivot can land on a hidden trip. Per-shot line-plot visibility never reaches the intersecter; a Kind::Lines pick hits hidden segments.
#549 Reset view frames hidden caves/trips/LiDAR — the model zooms out to include things you hid. boundingBox() unions every registered object with no visibility filter at all.

Design goal. Make the intersecter return correct visible-only results, so consumers (reset-view, Distance tool, lead occlusion) are written as if cwGeometryItersecter is simply correct — no visibility logic duplicated into each consumer. A committed failing test (testcases/test_cwRenderLinePlotPickVisibility.cpp, tagged [!shouldfail]) pins #575.

Phase 0 · How it works today current

Three layers, wired end-to-end. Visibility flows left→right; the intersecter is the one box the flow skips.

Phase 0a · Visibility is keyed on keywords

  • KeywordTab.qml:17-20 — KeywordVisibility { visibleModel: pipeline.acceptedModel; hideModel: pipeline.rejectedModel }. The filter pipeline is the source of truth for what's shown.
  • cwKeywordVisibility.cpp:105 resolveVisibility() — hide wins: if an object is in the reject model, hide it; else show. Fires on rowsInserted/Removed/modelReset.
  • cwKeywordVisibility.cpp:17 invokeVisible() dispatches by type: QQuickItem, cwRenderObject, or a generic setVisible(bool) — the last is how the proxy objects below get called.
  • Each keyword item carries an ObjectRole pointing at the object that knows how to hide the corresponding geometry at the right granularity.

Phase 0b · How the renderer honors visibility

  • Whole object — cwRHIObject::m_isVisible, synced in cwRhiScene.cpp:71 and skipped at gather in cwRhiFrameRenderer.cpp:569 (if (!object->isVisible() …) continue;).
  • Per-vertex centerline — cwRenderLinePlot::m_visibility is a QVector<quint8> (one byte/vertex, kVisible=0xFF/kHidden=0x00). cwRHILinePlot.cpp:44 uploads a parallel m_visibilityBuffer (uint attribute, binding 1); cwRHILinePlot.cpp:102 updateVisibilityBuffer() re-uploads on change; LinePlot.vert/.frag discard hidden vertices. This is the only place per-shot visibility exists.
  • Per-scrap — cwRhiTexturedItems stores per-id visible and skips hidden items at draw.

Phase 0c · What the intersecter does — and doesn't

  • Geometry is registered once as the full vertex list in cwRenderLinePlot.cpp:46-59 (setGeometry → addObject). Visibility toggles (setRangeVisible, .cpp:76-99) never call the intersecter.
  • The only visibility gate is whole render-object: cwGeometryItersecter.cpp:196-198 isPickable() = parent==nullptr || parent->isVisible(), hoisted once per sub-BVH at :951.
  • boundingBox() :652 unions all node boxes with no visibility check whatsoever — not even whole-object.
  • Key type is Key{cwRenderObject* parentObject, uint64_t id} cwGeometryItersecter.h:46; there is boundingBox(const QList<Key>&) :100 but no way to enumerate keys or filter by visibility.

Phase 1 · Whole-object visibleBoundingBox() interim

Ship the reset-view API immediately, correct for the common cases, before the deeper mask work.

  • Add cwGeometryItersecter::visibleBoundingBox() that unions node boxes for which node.Object.parent()==nullptr || parent->isVisible(). Main-thread only (bbox is computed against live Nodes).
  • Point cwBaseTurnTableInteraction::resetView() at visibleBoundingBox() (replacing boundingBox() at cwBaseTurnTableInteraction.cpp:331); keep the empty-box fallback.
Correct for: hidden LiDAR cloud, hidden scrap group, entirely-hidden centerline. Still wrong for: hiding a single cave/trip (the plot object stays visible). Reset-view is written once, against the final API — Phase 2 upgrades correctness with no change here.

Architectural note · put the seam in the right place

Today resetView() reads scene()->geometryItersecter()->boundingBox() directly (cwBaseTurnTableInteraction.cpp:331). "What bounds should the camera frame" is a scene / view-presentation concern, but cwGeometryItersecter::boundingBox() is an artifact of the pick-acceleration structure. Sourcing framing policy straight off the picking BVH leaks the picking subsystem's boundary into camera behavior.

Preferred seam: expose a scene-level framing-bounds accessor on cwScene / cwRegionSceneManager (e.g. framingBounds() / visibleFramingBounds()), and have resetView() (and any capture/export framing) call that. The visibleBoundingBox() work below then lives behind the scene seam rather than being one more method bolted onto the intersecter — the interaction depends on the scene for framing, not on the pick structure.

Blast radius: moderate — one new scene-level accessor, re-point resetView (and the Phase 1/2 work) at it; no data-model changes. Tracked as the architectural follow-up from the #549 review; can land before or alongside Phase 1.

Phase 2 · The per-primitive mask the fix

Give the intersecter its own visibility channel, fed from the same setRangeVisible calls the renderer already makes. Both picking and bounding box consult it. One mechanism → both #575 and per-trip #549 fall out.

2.1 · Mask on the Object

// cwGeometryItersecter.h — Object
QVector<quint8> m_visibility;   // empty = all visible (fast path)
const QVector<quint8>& visibility() const { return m_visibility; }
// helper: a Lines primitive at index i is visible iff both its
// endpoint vertices' bytes != kHidden (indices[i], indices[i+1]).

Empty mask ⇒ zero per-primitive cost for clouds, scraps, triangles, and any object that never partially hides.

2.2 · Mutation API (no BVH rebuild)

void cwGeometryItersecter::setRangeVisible(const Key& key, int start, int count, bool visible);
// Updates the mask on the live Node AND republishes the mask into the
// query snapshot (copy-on-write swap of a per-slot shared array).
// Geometry/BVH structure is unchanged, so this is O(range), not a rebuild.

2.3 · Render side pushes the mask

  • cwRenderLinePlot::setGeometry() — after addObject, seed an all-visible mask (or none).
  • cwRenderLinePlot::setRangeVisible() — additionally call geometryItersecter()->setRangeVisible(Key{this,0}, start, count, visible). The intersecter mask becomes the twin of the render m_visibility, fed by the same call.
  • (Optional, later) cwRenderTexturedItems per-id visibility can push analogously if scraps become pickable/framed at per-id granularity.

2.4 · Consumers honor the mask

  • Picking — in testPrimitive() (cwGeometryItersecter.cpp:1108+), after resolving the primitive's vertex indices, skip when the mask marks them hidden. Whole-object isPickable() stays as the cheap early-out.
  • Bounding box — visibleBoundingBox() unions only visible primitives. Masked objects iterate their primitives (can't use the cached node box); unmasked objects keep the fast whole-box path. O(vertices), once per reset.

2.5 · Thread-safety & the snapshot

The one real cost. Picks run off-thread against an immutable snapshot (SubBvhs). Bounding box runs on the main thread against live Nodes.
  • Bounding box: main-thread, reads the live mask directly — trivial.
  • Picking: the mask must live in the published snapshot so an in-flight pick sees a consistent view. Store the mask as a per-slot shared_ptr<const QVector<quint8>> alongside each SubBvh; setRangeVisible builds a new array and atomically swaps the pointer. BVH nodes are untouched, so toggling never triggers a rebuild and captured snapshots stay valid.
  • This mirrors the existing snapshot discipline (QuerySnapshot, slot invalidation) — the mask is just one more immutable-per-publish array.

Alternatives considered

  • Per-trip Objects — register each trip as its own Key and reuse whole-object isPickable. Rejected: restructures centerline registration, multiplies top-level BVH objects (perf), and changes the pick firstIndex contract (#530). The mask is purely additive — one object, one BVH.
  • Visibility callback into the render object — intersecter reads cwRenderLinePlot::m_visibility live. Rejected: couples the intersecter to render internals and breaks off-thread pick safety (the array mutates under the pick). The owned-copy-in-snapshot avoids both.
  • Fix only reset-view in the interaction (enumerate visible ranges, union in the interaction). Rejected: duplicates visibility logic per consumer and leaves #575 unfixed.
