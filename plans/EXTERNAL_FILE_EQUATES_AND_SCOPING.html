External File Integration — Equates & Station Scoping

Part of the External File Integration plan. Read the master plan and Phase 2 first. This file owns a foundational feature that is bigger than Phase 2 and underpins Phase 3: a persistent, user-editable equate (station tie-in) concept, the unified station-scope model it rides on, and opt-in native station prefixing. It was spun out of Phase 2 §15 because the equate work is a new model concept, not a Phase-2 bug fix.

Cross-cutting test rules (per-process safety, cavern serialisation, narrow tag filters, "existing tests always green") live in master §10.1 and apply to every commit here without restating.

0. Why this exists (and how it relates to Phase 2 & 3)

The user asked a deceptively simple question: how does a native survey tie into an attached external centerline? The answer, after tracing the code, is it can't — and the reason exposes a missing primitive that three separate features all need. This plan builds that primitive once.

  • Phase 2 is not gated by this work. Its only remaining blockers are §16 B2 / B3 / B6. But the note-station entry UI and the trip-page layout Phase 2 is finishing will be reshaped by the scope model here — so settling this design before hardening that UI avoids building it twice.
  • Phase 3 (cave-level attach) is already specified in terms of a stationPrefix on "Scope trips" (Phase 2 §11 stub, master §7.2 / §8.5.1 / §8.7). That is identical to the "native prefixing" designed here (§7 Path B). This plan makes the shared foundation explicit so Phase 3 consumes it rather than reinventing it (§6).
  • Phase 2 §15 F1 (external note-name / native-station collision) is a symptom of the same missing scope model and is folded into the scope lens (§4 commit 7) here.

1. Investigation findings (the current reality)

1.1 The four tie-in directions today

The scope a station lands in is set at export time in cwSurvexExporterCaveTask.cpp:104–125 (external trips get a *begin trip_<hex> wrapper; native trips are written unwrapped, :128–132), and mirrored on decode by cwLinePlotTask::scopedStationName (cwLinePlotTask.cpp:64–70).

TieWorks today?Mechanism
native → nativeYes Implicit — a shared bare name is one node. cwSurveyNetwork::addShot keys adjacency by cwStation::canonicalKey (lower-case only, cwStation.h:66), so two trips both naming 5 collapse to one solved point. cwFindUnconnectedSurveyChunks even defines connectivity as shared-name set-intersection (cwFindUnconnectedSurveyChunks.cpp:22–45).
native → externalNo cave_<hex>.Y vs cave_<hex>.trip_<hex>.X — different cavern scopes, distinct nodes.
external → nativeNo Same, reversed.
external → externalNo trip_<hexT1>.X vs trip_<hexT2>.X — different trip_ prefixes, distinct nodes even for identical tails.

Only native→native works, and it works by accident of naming, not by an explicit tie. The *begin trip_<hex> isolation wrapper (cwSurvexExporterCaveTask.cpp:98–103) is exactly what severs the other three.

1.2 There is no equate concept in the model at all

  • Nothing persists an equate. cavewhere.proto stores stations only by name — no equate/tie/alias/ connection field on Cave (:82), Trip (:107), Shot/StationShot (:217/:244), or ExternalCenterline (:76). FixStation (:65) is a georeferencing position, not a station-to-station tie. No runtime cwEquate/cwConnection class exists.
  • Import flattens equates. A Survex *equate A.1 B.1 is consumed at import: cwSurvexGlobalData::populateEquateMap (cwSurvexGlobalData.cpp:278–311) picks a canonical name and fixStationNames (:148–250) renames the stations to share it, then discards the map. The relationship becomes an implicit shared-name tie. (It also can't handle 3+-way equates — FIXME at cwSurvexGlobalData.cpp:292–296.)
  • Export synthesizes equates. The only *equate lines the exporter writes are for LRUD carrier stations, generated fresh from geometry each export (cwSurvexExporterTripTask.cpp:346–350, cwSurvexExporterRule.cpp:428–432) — not a round-trip of stored data.

Conclusion: a user-editable equate is a brand-new persistent model concept. The only reusable prior art is the mechanical *equate emission pattern and the import-side name-merge machinery.

1.3 Feasibility: decode is ready, emission has one real blocker

Decode round-trips an equate for free. Cavern emits both equated names at the same coordinate; the reader keeps both (cwSurvex3DFileReader.cpp:89–105); and splitLookupByCave keys the per-cave lookup by regex capture 2 (cwLinePlotTask.cpp:455–456, cwCavernNaming.cpp:65–67), producing two distinct cave-local keys sharing one position with no collision. An equate is literally "two aliased names, one point," and the pipeline already represents it that way.

Emission is a bare *equate line — SPIKED & RESOLVED 2026-07-22 (commit 1, testcases/test_cwEquateSpike.cpp). The feared export barrier does not apply to our driver. Cavern's error 26 ("Station not exported") fires only when a station was *export-ed but not to a deep enough level: the check at readval.c:329 runs only when min_export is neither 0 nor USHRT_MAX, and a never-exported station keeps min_export == 0, making the check a no-op. The survex corpus shows exactly this asymmetry: survex/tests/export.svx passes because inner.1 is *export-ed at both nesting levels; survex/tests/exporterr1b.svx fails because it is exported at the inner level only — an incomplete chain, not a missing one. Since our driver emits no *export at all (cwSurvexExporterCaveTask.cpp:104–125), a bare qualified *equate just resolves. No *export and no *infer exports are needed. The spike drove real cavern solves and confirmed coincident positions at cave scope, region scope, and for an N-way (3-operand) equate. This materially simplifies commits 4 and 5: emission is one *equate line at the tie's scope.

Two lesser wrinkles to design around, not blockers:

  • Topology stays "two nodes at one point." The .3d line-endpoint index is first-label-wins (cwSurvex3DFileReader.cpp:98–104), so the alias name gets a position but may carry no adjacency edges; updateCaveNetworks (cwLinePlotTask.cpp:576–620) does not merge the two cave-local keys. Positions are perfect; consumers must treat the aliased keys as position-aliases, not assume one merged node.

2. Decisions locked (2026-07-22)

#DecisionRationale
D1Alias, never merge. A 3D station location may carry multiple names; both stations keep their identity. Mandatory for external (can't rename a foreign file's stations) and matches the decode's natural representation (§1.3). The import's rename-and-drop is the wrong model for a persistent tie.
D2Both cave-scoped and cross-cave / region-level equates ship in this plan. Cross-cave is a second storage home + a second emission path (§5), not a new model — the handle (D6) is shared. Two caves that physically connect (a dig breaks through) is a first-class case, not a someday one; the self-contained handle makes it cheap to include now.
D3Spike first — DONE 2026-07-22 (GREEN). Commit 1 proved cross-scope *equate solves at both cave and region scope with a bare line: no *export, no *infer exports. The export barrier was a false alarm (see §1). Emission in commits 4/5 is one *equate line. Small, decisive, and the whole model is worthless if cross-scope *equate can't solve. It can.
D4N-way equates. An equate is a set (≥2) of station handles; fix the import's 2-way limit while here. A junction where three surveys meet is one physical point, not two pairwise ties.
D5Additive / opt-in. native→native keeps working implicitly by shared name (backward compat). Equates are added for the three external directions and for prefixed-native trips. No migration forced on existing flat projects.
D6Self-contained station handle (scope, containerId, tail) with containerId always populated (option A): caveId for a NativeCave station, tripId for a Trip-scoped (external / Scope-trip) station. Location-independent identity — the same handle works on a cave or the region, so cross-cave needs no schema change. A tripId (not a denormalized caveId) for scoped stations keeps a re-parented trip correct.

3. Foundation: one unified station scope

Three things scope a trip's stations, and they are the same shape. The whole plan hinges on collapsing them into one accessor so equates, the scope lens, native prefixing, and Phase 3 all speak it.

Trip kindScope prefixSource
Native (flat) (empty)—
External (trip attach)trip_<hex>. externalCenterline
Scope trip (Phase 3 cave attach) / native prefixed <stationPrefix>. stationPrefix
QString cwTrip::scopePrefix() const {
    if (!externalCenterline().isEmpty()) return cwCavernNaming::tripScopePrefix(id()); // "trip_<hex>."
    if (!m_stationPrefix.isEmpty())      return m_stationPrefix + QLatin1Char('.');     // "A."
    return QString();                                                                   // flat native
}
bool cwTrip::isScoped() const { return !scopePrefix().isEmpty(); }

The three cwTrip accessors (solvedStationPositions/solvedNetwork/ solvedStations, cwTrip.cpp:406–503) currently branch on m_externalCenterline.isEmpty() and strip tripScopePrefix(id()). They switch to branching on isScoped() and stripping scopePrefix() — the strip logic is already there, just generalised. This subsumes Phase 2 §15 R2 ("shared externallyBacked predicate") and A1 ("uniform external-station accessor"): both C++ (cwLinePlotGeometry.cpp, cwLinePlotLabelView.cpp) and QML (TripPage.isExternal, NoteStation / NoteLiDARStationItem validator bindings) sites that hand-roll externalCenterline.entryFile.length > 0 collapse onto isScoped.

The failure mode to avoid: building equate emission (or any new scope logic) around the external-only trip_<hex> prefix. That works for trip-attach but silently breaks for Phase-3 Scope trips. Everything must go through scopePrefix().

4. Commit sequence

Commit 1 — Equate spike (throwaway) — DONE 2026-07-22, GREEN

Result: cross-scope *equate solves at both scopes with a bare *equate line — no *export, no *infer exports. The export barrier was a false alarm (see §1, "Emission is a bare *equate line"). The spike (testcases/test_cwEquateSpike.cpp, tag [EquateSpike]) drove real cavern solves via cwCavernRunner::run and read positions back with cwSurvex3DFileReader, asserting coincident coordinates for:

  • Cave scope — two isolated *begin trip_<hex> blocks inside one *begin cave_<hex>, equated by cave-relative name. ✓
  • Region scope — two *begin cave_<hex> blocks inside the region wrapper, equated by fully-qualified cave_<hex>.<tail> name. ✓
  • N-way — one *equate with three qualified operands lands all three at one coordinate, so an N-way tie is a single emitted line, not a pairwise chain (validates D4). ✓

Emission point for commit 4 (cave): cwSurvexExporterCaveTask::writeCave, a new slot between the trip loop end (cwSurvexExporterCaveTask.cpp:133) and the closing *end (:135), operands cave-relative. Emission point for commit 5 (region): cwSurvexExporterRegion::exportRegion, after the cave loop (cwSurvexExporterRegion.cpp:58) and before the outer *end (:60), operands fully qualified cave_<hex>.<tail>. The spike file is throwaway — delete it once those two emission points land.

Follow-up surfaced by the spike (feeds commit 6 UX, not a blocker): equating the two directly fixed stations of two independently-fixed subnetworks raises cavern error 46 ("Station already fixed or equated to a fixed point"). The safe tie is between non-fixed stations of each component (each component keeps its own *fix elsewhere), which cavern solves as an over-constrained loop and reports as misclosure — the exact signal commit 6 surfaces to the user. The equate UX must therefore steer the user to tie ordinary stations, and route error 46 to a friendly message rather than a raw cavern failure.

Commit 2 — cwTrip::scopePrefix() foundation + predicate

Add m_stationPrefix (plain member + explicit setter + NOTIFY, no bindable), scopePrefix(), isScoped(). Re-point the three solved-* accessors and the R2/A1 consumer sites. Proto gets Trip.station_prefix (string, empty default). No behaviour change yet for existing projects (empty prefix ⇒ identical output). Tests: the existing external round-trip stays green; add a native-trip-with-prefix accessor test.

Commit 3 — Equate model + persistence (self-contained handle)

An equate is an N-way set of self-contained station handles (D6): 

struct cwStationHandle {
    enum Scope { NativeCave, Trip };
    Scope   scope;
    QUuid   containerId;   // caveId when NativeCave, tripId when Trip
    QString tail;
};

containerId is always populated — auto-filled from the owning cave for a within-cave native tie, so the UI never asks for it. Because the handle is location-independent, one type serves two homes:

  • cwCave equate list — within-cave ties. Validation: every Trip handle's parentCave and every NativeCave handle's containerId must equal the owner cave. Travels with the cave.
  • cwCavingRegion equate list — cross-cave ties (two caves that connect). Handles may span caves; no owner-cave constraint.

Proto: repeated Equate on both Cave and the region message, each with repeated StationHandle { Scope scope; bytes container_uuid; string tail; }. Runtime objects with undo, save/load, equatesChanged. Fix the import's 2-way limit (D4). No solve wiring yet.

A Trip handle covers external trips (trip_<hex>) and Phase-3 Scope trips (stationPrefix) with no distinction — emission renders each via trip->scopePrefix(). A NativeCave handle covers bare native stations and the cave-level-externalCenterline variant (stations directly under cave_<hex>. with no trip).

Commit 4 — Cave-level equate emission + decode consumption

Emit each cwCave equate as a *equate at cave scope — the natural slot is after the trip loop closes, before *end cave (cwSurvexExporterCaveTask.cpp, between the :133 loop end and the :135 *end), where every native station and every *begin trip_<hex> sibling is in scope. Operands render cave-relative: <scopePrefix><tail> for a Trip handle, <tail> for a NativeCave handle. No *export/*infer line is needed (commit 1). Make consumers treat the aliased cave-local keys as position-aliases (the §1.3 wrinkles): confirm geometry + labels resolve through the shared position; document that the cave network holds two nodes at one point. Test: red→green — a native station equated to an external tail draws them coincident (fixture with a deliberate pre-tie gap).

Commit 5 — Region-level (cross-cave) equate emission

Emit each cwCavingRegion equate at region scope in cwSurvexExporterRegion, after both *begin cave_<hex> blocks are declared as siblings. Operands render fully qualified: cave_<hex>.<scopePrefix><tail> for a Trip handle (cave hex from trip->parentCave()), cave_<hex>.<tail> for a NativeCave handle (cave hex from containerId). No *export/*infer line is needed (commit 1). Decode is unchanged — splitLookupByCave already routes each qualified name to its cave and keys by the tail, so a cross-cave equate lands as two position-aliased keys, one in each cave's lookup. Test: red→green — a station in cave A equated to one in cave B solves coincident across the cave boundary.

Commit 6 — Equate UX (the pain-point payoff)

Survex's pain is a hand-managed text list of qualified names. CaveWhere has the stations and positions, so make it visual:

  • Create by picking two stations, never typing qualified names — select a station → "Tie in…" → pick the partner via the scope search picker (commit 7) or by clicking it in the 3D view. The un-typeable-UUID problem disappears because you select.
  • Existence + misclosure feedback — confirm both stations exist (today's validator is character-only, §1.2) and surface the gap distance cavern already computes ("these were 0.4 m apart").
  • A "Ties" panel — friendly rows (Trip A · 5 ↔ Whitmore.svx · X), add/remove, misclosure per row. A reviewable table, not a text file. Shows both a cave's own ties and the region's cross-cave ties (§5).
  • Deferred nicety: auto-suggest an equate when two absolutely-fixed components land within ε.

Commit 7 — Scope lens (reference / autocomplete / prefix-free typing)

Distinct from equate (a join that affects the solve): the scope lens is reference (look up where a station is) for note/scrap/LiDAR entry.

  • Generalise cwScopeStationListModel (today external-only, prefix-matched against the region network, cwScopeStationListModel.cpp:76–99; wired only in ExternalCenterlineTripPanel.qml:105–111) to be trip-driven from trip->solvedStations(), so native and external get the same autocomplete. Default scope = the current trip.
  • Wire the scope model + a scope selector into NoteStation.qml and NoteLiDARStationItem.qml (shared StationDoubleClickTextInput + StationValidator ⇒ one change). Type a2 = "a2 in this trip"; expand to This Cave / another trip when you need a tie-in reference.
  • Folds in Phase 2 §15 F1. Membership feedback surfaces an out-of-scope name ("a2 isn't in this trip — tie into <cave> a2?") instead of the current silent anchor to a colliding native station. Membership stays a non-blocking advisory (a separate signal, not folded into cwStationValidator::validate(), which must stay permissive).
  • Deferred follow-up (its own item): a searchable, whole-cave station picker for big caves (many myprefix1.1, myprefix2.1), optionally ranked by 3D distance to the trip (solvedStations() carries positions).

Commit 8 — Native prefixing, Path A (dotted names)

Widen the native cwStationValidator grammar to allow . (currently native forbids it, cwStationValidator.cpp:20; only external allows dot+space, :26). Then A.1 is simply a station name, cave-wide identity as typed, ties by full name like any native name today. The scope lens (commit 7) makes typing ergonomic (type 1, pick A.1) and can group by the common prefix. Nearly free, no solve change, no migration hazard, and it supports the real myprefix1.1 caves immediately. Safety check: dots decode cleanly (stationRegex capture 2 is \S.*; native has no trip_<hex>. to confuse them with).

Commit 9 — Native prefixing, Path B (declared stationPrefix)

The ergonomic + structural layer, shared with Phase 3: a trip carries a declared stationPrefix (from commit 2); chunks store bare tails; the qualified identity is prefix.tail, applied at the same layer external trips are wrapped (cwSurvexExporterCaveTask.cpp:104–125 generalised from trip_<hex> to any scope label). Cross-trip ties then use equates (commits 3–6).

Migration hazard to surface in the UI: adding a prefix to an existing native trip renames its stations (5 → A.5), silently disconnecting any implicit shared-name ties it had to other trips. Make the consequence visible; optionally offer to convert the broken implicit ties into explicit equates on opt-in.

5. Cave-owned vs region-owned equates

Both ship in this plan (D2). They differ only in storage home and emission scope; the self-contained handle (D6) is identical, so cross-cave adds no new field — a region-owned equate is simply one whose handles resolve to more than one cave.

Cave-owned (cwCave) Region-owned (cwCavingRegion)
Covers native↔native (explicit), native↔external, external↔external — within one cave two caves that physically connect
Handle constraint all handles resolve to the owner cave (validated) handles span caves; none
Emitted by cwSurvexExporterCaveTask, inside *begin cave, cave-relative operands (commit 4) cwSurvexExporterRegion, region scope, cave_<hex>.-qualified operands (commit 5)
Travels with the cave (part of its definition) the region

A within-cave native↔native explicit equate (tying two differently-named native stations without renaming either) is supported by the cave-owned list — the implicit shared-name tie (D5) stays the default, the explicit equate is additive.

6. Phase 3 compatibility

Phase 3's cave-level attach auto-creates Scope trips under a cave, each carrying a stationPrefix (the file's survey designation) rather than a per-trip externalCenterline (Phase 2 §11 stub: trip.stationPrefix "Phase 3 adds it"; isScope/scopeHeaderComp; master §7.2 / §8.5.1 / §8.7). Because this plan builds on cwTrip::scopePrefix() (§3) and handles reference trips (§4 commit 3):

  • An equate to/from a Phase-3 Scope trip works by construction — it's just a trip with a non-empty scopePrefix().
  • external→external within a cave-level attach (two Scope trips of one file) is the same case, no special handling.
  • The isScoped() predicate (commit 2) is exactly what Phase 2 §15 R2 says the relaxed-grammar / externally-backed check must become, because "Scope trips carry stationPrefix, not externalCenterline."
  • Path B (commit 9) is the Phase-3 Scope-trip mechanism — Phase 3 reduces to "a cave attach that creates Scope trips + equates for the joins," not a new naming system.

7. Risks

RiskMitigation
Cross-scope *equate can't be made to solve. RETIRED — commit 1 spike proved it solves at both scopes with a bare line. Was the gate; now green.
"Two nodes at one point" leaks into topology consumers (loop-closure display, neighbour walks) that assume one merged node. Commit 4 audits the consumers; document the position-alias contract; add a test that walks the network from the alias side.
Path-B prefixing silently disconnects an existing survey. Opt-in only; UI surfaces the rename consequence; offer implicit→ explicit-equate conversion (commit 8).
Region-scope *equate hits a deeper export barrier than cave scope. RETIRED — there is no export barrier (commit 1); region scope solved identically. The self-contained handle (D6) already carries the cave id, so cross-cave needs no schema change.
