5. Scanner edge cases (Survex)

  • Cycle detection. Walk the include closure with a visited set keyed by canonical absolute path (QFileInfo::canonicalFilePath). On a re-visit, append a warning "circular include: a.svx → b.svx → a.svx, stopping" to ScanResult.warnings and stop expanding that branch. Don't fail the scan — the rest of the closure is still useful. No depth cap — the canonical-path visited set is the only thing that can runaway-loop, and we already catch it. A legitimately deep tree (50+ levels) is unusual but valid and should just go through.
  • Case-sensitivity. The scanner tries each *include target as-written first. If the file doesn't exist on a case-sensitive filesystem (Linux), it retries with a case-insensitive directory listing (QDir::entryList, lowercased compare). On success via the fallback, emit a warning "include 'Foo.svx' matched 'foo.svx' — consider renaming to match". This lets a project authored on macOS open on Linux without surprise. Hard miss after the fallback → warning, dependency not added.
  • Encoding. UTF-8 with BOM detection; fall back to Latin-1 on UTF-8 decode failure (survex historically tolerated this). Track which encoding the fallback selected so the warning can name it.
  • Line continuations and comments. Standard survex syntax: ; begins a comment; directives that span multiple lines use the same mechanism Survex itself does (no special continuation char — *include is a single line). The scanner only needs to find *include "path" tokens; *begin / *end nesting doesn't change include resolution.

6. Scanner edge cases (Compass & Walls)

  • Compass .mak grammar. Reuse the existing Compass importer's lexer (cwCompassImporter — verify it tolerates .mak-only parsing without pulling shot data). A bare .dat returns just itself.
  • Walls .wpj coverage. dewalls handles the WPJ format, but has not been exercised against real-world projects at CaveWhere's scale — that's the Walls acceptance gate in §3. If a real .wpj reveals dewalls gaps, push Walls to Phase 1.5 with Survex + Compass shipping first. Don't attempt to write a WPJ parser from scratch.
  • formatFor(entryFile) dispatches by extension (case-insensitive QString::endsWith). Mapping: .svx → Survex; .dat, .mak → Compass; .wpj, .srv → Walls. No extension → Monad::Result error "cannot determine format from extension"; the attach dialog surfaces this and [Attach] stays disabled (per master §8.8 question 18).

7. Cross-format includes (Survex only)

Cavern itself supports including non-Survex files from a .svx (*include "compass.dat" works at solve time because cavern auto-detects format). Our scanner mirrors that: scanSurvex dispatches each *include target through formatFor() on the included file's extension and recurses via the matching per-format scanner.

  • *include "foo.svx" (or no extension → auto-append) → recursive scanSurvex.
  • *include "foo.dat" or "foo.mak" → scanCompass; results merged into the parent ScanResult (dependencies appended, warnings appended).
  • *include "foo.wpj" or "foo.srv" → scanWalls; same merge semantics.
  • Cycle detection spans formats. The canonical-path visited set is shared across the entire walk, so a .svx → .mak → .svx chain that loops back catches the same way a pure-.svx cycle does.
  • Reverse direction is not supported by the file formats themselves. Compass .mak only references .dat; Walls .wpj only references .srv and .wpj. Neither Compass nor Walls has a directive that pulls in a Survex file, so scanCompass and scanWalls stay in-format. If a user wants to mix, they make a Survex driver the entry file and *include outwards from there.
  • The pulled-in foreign files become first-class project dependencies — the reconcile step (master §5.2) copies them into the owner's external-centerline/ subdir alongside the entry .svx, with their relative directory structure preserved so the entry's *include paths still resolve.

8. Cancellation semantics

  • Scan cancel (Cancel button in the attach dialog, master §8.8 question 14). Scan runs via QtConcurrent::run; the scanner checks QFuture::isCanceled() between each *include resolution. Nothing has been persisted yet, so cancel just discards the partial result.
  • Detach during an in-flight solve. Detach mutates the data model + enqueues a cwSaveLoad remove-tree job. The data-model change emits a signal that hits the existing cwLinePlotManager trigger, which starts a new solve via the Restarter. The Restarter cancels the in-flight future (per plans/LINEPLOT_ASYNC_PORT_RESEARCH.html Q6: the cancel today is “discard result”, not in-flight interrupt), so the old solve runs to completion and its result is dropped on return. The new solve runs without the detached attachment. Latency is bounded by the lineplot port's eventual Q4 work (cooperative QPromise-based cancel), not blocking for v1.
  • Reconcile job in flight at detach. cwSaveLoad jobs run to completion regardless of UI state. The reconcile batch's final .context(this, …) guards against “attachment removed before reconcile finished” by re-checking the attachment exists in the data model before applying any post-reconcile state (e.g. updating the watch set). Half-written copies never happen because each copy is atomic at the FS level (write-to-temp + rename via QSaveFile).

9. cwSaveLoad job-queue audit (first commit)

Before any reconcile code lands, audit the existing cwSaveLoad API to confirm coverage for the three operations reconcile needs. Output of the audit goes in the first Phase 1 commit message:

  • copyIfNewer(src, dst) — verify Action::Copy's contract: does it overwrite unconditionally, or compare (mtime, size) first? If unconditional, wrap as Action::Custom with the mtime/size precondition.
  • removeFile(path) — Action::Remove with Kind::File; verify single-file semantics.
  • removeTree(path) — Action::Remove with Kind::Directory; verify recursive behavior. If non-recursive, wrap as Action::Custom calling QDir::removeRecursively.
  • Rename + delete cascade through the owner dir. The plan's "for free" claim (master §5.1) needs verification on a synthetic fixture with an external-centerline/ subdir already in place. Failing case → add the subdir to the rename/remove child-list (one extra entry in the existing handler, not a new code path).
  • Dir-helper signature. The two-overload sketch in master §5.1 (externalCenterlineDirHelper(const QDir&) both with the same arg type) won't compile. Consolidate to a single externalCenterlineDirHelper(const QDir& ownerDir); the public externalCenterlineDir(cwCave*) / (cwTrip*) entry points dispatch.

10. Driver *include path generation

  • Use absolute paths with forward slashes. Relative paths via QDir::relativeFilePath produce long ../../../ chains since the driver scratch dir is in /tmp (or Windows %TEMP%) while the project tree lives elsewhere. Absolute is simpler and just as portable for the driver's one-solve lifetime.
  • Always emit forward slashes. Survex *include accepts forward slashes on every platform, including Windows (C:/Users/…). Do not call QDir::toNativeSeparators.
  • Always quote. Sanitised cave/trip dir names may contain spaces; Survex requires quoting in that case and tolerates quoting otherwise.

11. Watcher per-platform notes

  • macOS: QFileSystemWatcher uses FSEvents; reliable for ordinary writes. Atomic-rename saves (some editors, e.g. vim with :wq) emit removed-then-created. Our scanner re-runs on any change in the watched set, so the rename pattern still drives a re-solve.
  • Linux: inotify. Default user watch limit is 8192 (fs.inotify.max_user_watches). v1: not addressing — the typical attachment is well under that. v1.x can fall back to polling on overflow.
  • Windows: ReadDirectoryChangesW; reliable for local disks, unreliable for network shares. Already in master §11 risk table; manual Reload covers the gap.
  • Symlinks: follow once at scan time (QFileInfo::symLinkTarget); watch the canonical target. Don't add symlink-loop detection separately — the canonical-path visited set in cycle detection (§5) catches it.
