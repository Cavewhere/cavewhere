Implementation order: Phase 4 ships after Phase 5 (orphan detection) — see the master plan §9 implementation-order note. Phase numbers are stable names, not the delivery sequence.
Prerequisite refactor — per-format strategy split in cwExternalCenterlineScanner. The scanner already carries three per-format axes side by side (dependency walkers, metadata line-parsers, per-format regexes/comment strippers) with format-dispatch switches duplicated in scanByFormat and parseSeededMetadata. Phase 4/5 station extraction from external files adds a fourth axis. Before implementing it, restructure into per-format units (a FormatTraits-style strategy, or per-format TUs — cwSurvexFormat / cwCompassFormat / cwWallsFormat — behind the existing namespace facade) so the duplicated switches collapse and the file stops growing monolithically. Pure mechanical restructure, zero behavior change; whichever of Phase 5 / Phase 4 ships first does it.

What this file will contain (after the design pass)

  • Data model — new cwEquate protobuf message with two fully-qualified station references; persistence on cwCave (Phase 4) and on cwCavingRegion (Phase 6); equality / hash for change detection.
  • Editor UI — where the editor lives (CavePage sub-page? new top-level page? dialog?), row layout, station picker behavior (autocomplete against the resolved network, including across Scope trips), delete affordance.
  • Re-solve plumbing — signal flow from equate edits into cwLinePlotManager::Restarter; equate set as part of the worker input snapshot for deterministic driver emission.
  • Scope-trip interaction — equating two stations inside the same cave-level attach but in different Scope trips; picker UI exposing stations from sibling Scope trips.
  • Conflict UX — what happens when an equate contradicts cavern's loop closure (large traverse error after solve), and how the user sees it on CavernOutputPage.
  • Station rename / disappearance — orphaned equates after the external file's station naming changes; coordination with Phase 5 orphan detection.
  • Per-commit test gates and fixture files.
