Implementation order: Phase 5 ships before Phase 4 (equates) — station renames in the user's native tool are the daily workflow, and orphan detection is what keeps them from silently breaking scrap references. See the master plan §9 implementation-order note.
Prerequisite refactor — per-format strategy split in cwExternalCenterlineScanner. The scanner already carries three per-format axes side by side (dependency walkers, metadata line-parsers, per-format regexes/comment strippers) with format-dispatch switches duplicated in scanByFormat and parseSeededMetadata. Phase 5's station extraction from external files adds a fourth axis. Before implementing it, restructure into per-format units (a FormatTraits-style strategy, or per-format TUs — cwSurvexFormat / cwCompassFormat / cwWallsFormat — behind the existing namespace facade) so the duplicated switches collapse and the file stops growing monolithically. Pure mechanical restructure, zero behavior change; since Phase 5 ships before Phase 4, this refactor lands here (Phase 4 inherits it).

Phase 5 — Orphan detection

What this section will contain

  • Diff mechanism — after each solve, build the set of station names referenced by notes/scraps per cave and intersect against the resolved network's station name set; non-intersecting names are orphans.
  • UI surface — orphan list in a new CavernOutputPage section (or its own page?), per-row actions ("remap to…" autocomplete vs "delete reference"), how bulk orphans are presented (a 200-orphan list shouldn't drown the page).
  • Data model — do orphans persist between solves or rebuild from scratch each time? My current guess: rebuild per solve, but cache the last user dismissal so the same orphan doesn't re-surface after a no-op re-solve.
  • Native-only project applicability — the same mechanism works on native-only projects (a renamed-station case); document the v1 behavior for that audience.
  • Per-commit test gates and fixture files.

Phase 5b — Replace / Re-link attachment

What this section will contain

  • Re-link flow — banner trigger (watcher + startup-probe per §8.8 question 16); picker dialog; prefix/station-set validation against the cached in-project copy; local_settings.json update.
  • Replace flow — picker dialog; diff dialog (currently sketched in the master); Kept / Orphaned / New classification; auto-create-on-new toggle; orphaned-trip handling (kept with error badge, or deleted outright).
  • Both flows wire into the ExternalCenterlineTripPanel's MissingSourceBanner sub-component (Phase 2 ships the banner stub; Phase 5b wires its Re-link button to ReplaceCenterlineDialog).
  • New cwExternalCenterlineSync::replaceSource() / relinkSource() APIs.
  • New cwExternalCenterlineScanner::diff() helper that emits the Kept/Orphaned/New classification.
  • Per-commit test gates and fixture files.
