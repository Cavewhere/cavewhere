Detect Outlier Fix-Station Coordinates

Find fix stations whose coordinate is a data-entry error (wrong CS, wrong UTM zone, transposed digits) — the class of mistake that silently blows up the whole 3D render — and surface it where the user can act on it. Grows out of #596 (removing the confusing "Recenter world origin" button).

Origin of this work. #596 asked "what does Recenter world origin do, and should we remove it?" It manually recentered the world origin — an internal float-precision offset — with no visible effect, so it was removed (done menu item deleted from DataMainPage.qml, test renamed). That button was a manual knob for a problem the app should detect on its own: a coordinate so wrong it breaks rendering. This plan builds that detection.

2. Problem & precision math

CaveWhere subtracts a world origin from every coordinate before it reaches the GPU, so that survey data (UTM coordinates in the millions) fits inside float (32-bit) precision in the shaders. The origin is auto-computed on the first line-plot solve as the centroid of all fix stations (cwLinePlotManager.cpp:691 → cwCavingRegion::recomputeWorldOrigin()).

A float has a 23-bit mantissa → relative precision 2⁻²³ ≈ 1.19e-7. Absolute error at coordinate magnitude d is d × 1.19e-7. Choosing a 1 cm budget:

d_max = 0.01 m / 1.19e-7 ≈ 84,000 m  ≈ 84 km

So any rendered vertex more than ~84 km from the origin has >1 cm of jitter. A single typo'd fix — wrong UTM zone or a transposed digit — lands hundreds of km out. Because the origin is the mean, that one point drags the origin off the real data and inflates the scene bounding box, so the auto-framed 3D view shows the real cave as a sub-pixel dot. The user sees an empty or garbage render with no explanation.

Not a generic "your survey is too big" warning. A region that legitimately spans >84 km can't be helped by recentering, and warning about it on every solve is unactionable noise. We detect only the isolated outlier — a fix far from the cluster of every other fix — which is almost always a typo and is fixable. The relative test (§4) naturally ignores legitimately spread-out surveys.

3. Altitude: a dedicated validator, owned by the region

"Outlier" is defined relative to the whole survey, so detection needs every cave's fixes at once — a region-scoped computation. But cwCavingRegion is already the app's busy root model, and this work is a distinct responsibility (gather + reproject, median math, per-cave cwError lifecycle, a summary property, signal wiring). So it lives in a new class, cwFixStationValidator, owned by the region rather than baked into it.

The region keeps only what is genuinely its own state — writing the world origin to geoReference — and delegates everything else. The validator gathers each cave's fixes, reprojects into the region's global CS (the reprojection logic that recomputeWorldOrigin() does today moves here), classifies, and attributes any flag back down to the owning cave's existing errorModel().

New class — cwFixStationValidator QObject

class cwFixStationValidator : public QObject {
    Q_OBJECT
    Q_PROPERTY(QString warningMessage READ warningMessage NOTIFY warningMessageChanged)
    Q_PROPERTY(int outlierCount READ outlierCount NOTIFY outlierCountChanged)
public:
    explicit cwFixStationValidator(cwCavingRegion* region);   // owned by the region

    struct FixCandidate  { cwCave* cave; QUuid fixId; cwGeoPoint global; };
    struct Classification { QList<FixCandidate> inliers; QList<FixCandidate> outliers; };

    // Pure math — unit-testable with hand-built candidates, no region needed.
    static Classification classifyCandidates(const QList<FixCandidate>& candidates);

    Classification currentClassification() const;      // gather + reproject + classifyCandidates
    std::optional<cwGeoPoint> robustWorldOrigin() const;// inlier centroid; consumed by the region

    QString warningMessage() const;                    // render-view summary ("" when clean)
    int outlierCount() const;

public slots:
    void revalidate();          // recompute + append/remove cwError on each cave

signals:
    void warningMessageChanged();
    void outlierCountChanged();

private:
    cwCavingRegion* m_region;
    QHash<cwCave*, cwError> m_caveWarnings;   // stored per-cave error → remove/update in place
    // owns its own connections to caves' fixStations signals + geoReference CS changes
};

Region side stays tiny:

// cwCavingRegion
cwFixStationValidator* fixStationValidator() const { return m_fixStationValidator; }

void cwCavingRegion::recomputeWorldOrigin() {                 // thin consumer, unchanged signature
    if (auto origin = m_fixStationValidator->robustWorldOrigin())
        m_geoReference->setWorldOrigin(*origin);
}
One validator pass feeds a per-cave cwError; the cave-list badge and CavePage read that model directly, the render overlay reads the validator's summary property.

The per-cave cwError lifecycle copies the proven pattern from cwTripCalibration.cpp:329-360 (the declination warning): keep the current cwError per flagged cave, append() it, setData(MessageRole) for in-place text updates, and remove() when the coordinate is corrected — with a stable errorTypeId so it is identifiable and user-suppressible. Severity is Warning (survex still runs; warnings are suppressible — the escape hatch for a genuinely remote fix).

4. Detection algorithm

Working in the region's global CS, on the reprojected candidate points:

  1. Robust center c = component-wise median of all candidates.
  2. Robust scale s = median of the distances |pᵢ − c| (a typo-proof cluster radius).
  3. Flag candidate i when |pᵢ − c| > max(kOutlierFloorMeters, kMadMultiplier × s).
  4. Run only when there are ≥ kMinFixesForDetection fixes (need a cluster to be an outlier from).
Why the max() of two thresholds. A flagged point must exceed both:
  • the precision floor (~84 km) — below it, rendering is fine, so nothing to flag; and
  • k × cluster-radius — in a legitimately spread survey the radius s is large, the threshold rises, and nothing is flagged. Only a point far from an otherwise tight cluster trips it.
This is precisely the discriminator that separates "typo" from "big survey."
The outlier trips both thresholds; the good cluster trips neither. Two legitimate caves 100 km apart would form two clusters — a large s, nothing flagged.

This math is the pure static cwFixStationValidator::classifyCandidates() from §3 — no region, no side effects, so the threshold logic is unit-testable with hand-built candidates. currentClassification() wraps it with the gather/reproject step. Both recomputeWorldOrigin() (via robustWorldOrigin()) and the per-cave attribution consume the same classification — one gather, two readers.

5. Robust world origin behavior change

Today recomputeWorldOrigin() sets the origin to the mean of all fixes. The mean is not robust — one outlier drags it, so the good geometry ends up far from the origin and the whole scene jitters / frames wrong. Fix: compute the centroid from inliers only (equivalently, the median). Then the origin sits in the real cluster, the good geometry renders crisp and centered, and only the bad station flies off — which is the behavior we want for diagnosis, and what makes the render-view banner meaningful.

Touches tested behavior. test_cwCavingRegion_worldOrigin.cpp has "averages multiple fixes" asserting the mean. For clusters with no outlier the inlier-centroid equals the old centroid, so that test is expected to still pass; a new test asserts an outlier no longer drags the origin. Isolated as its own commit so it can be dropped if you'd rather keep mean semantics — at the cost of the render-view banner being far less useful (scene stays garbage rather than showing one station flown off).

7. Phased delivery (commits)

Commit 1 — cwFixStationValidator detection core (C++, no UI)

  • new cavewherelib/src/cwFixStationValidator.h/.cpp — the class from §3, registered in CMakeLists.txt. This commit lands the pure parts: FixCandidate, Classification, static classifyCandidates() (median center + scale + threshold), and currentClassification() (gather + reproject, copied from recomputeWorldOrigin — the region's own copy is deleted in Commit 2, so the loop is duplicated for exactly this one commit). No attribution, no property, no signal wiring yet.
  • new constants in an anonymous namespace in the .cpp (see §8).
  • edit cwCavingRegion.h/.cpp — own the validator (new cwFixStationValidator(this)), expose fixStationValidator() + Q_PROPERTY. recomputeWorldOrigin() untouched this commit (still its own centroid).
  • new testcases/test_cwFixStationValidator.cpp — tight cluster → no outlier; one typo → flagged with correct cave/fix; two legit clusters → none; < min fixes → none; reprojection across differing CS. Pure-math cases hit classifyCandidates() directly.

Commit 2 — Robust world origin (C++) behavior change

  • new cwFixStationValidator::robustWorldOrigin() — inlier centroid (std::optional, empty when no candidates).
  • edit cwCavingRegion::recomputeWorldOrigin() — collapses to the thin consumer from §3; its old gather/centroid is deleted (now lives in the validator).
  • edit test_cwCavingRegion_worldOrigin.cpp — confirm no-outlier cases unchanged; add "outlier does not drag origin".

Commit 3 — Attribute warnings to cave errorModel (C++)

  • new cwFixStationValidator::revalidate() — run currentClassification(), then per cave append/remove/setData a cwError(Warning) with a stable errorTypeId and a message naming the station and its distance. Store the current cwError per cave in m_caveWarnings (cwTripCalibration pattern).
  • new self-owned trigger wiring in the validator — connect to each cave's fixStations dataChanged/countChanged/modelReset and the region's cave add/remove + geoReference CS-change signals, calling revalidate(). Region-independent of the origin gate, so it re-runs on any fix-coordinate edit.
  • new tests — SignalSpy on cave.errorModel().warningCount; warning appears on typo, clears on correction, updates in place on edit. Lights up the cave-list badge for free.
Review before implementing — empty global CS. When geoReference()->globalCoordinateSystem() is empty, gatherCandidates() takes every fix's raw coordinate with no reprojection target (cwFixStationValidator.cpp:151-153), so two caves whose fixes use different input CSs would be compared as raw eastings and a legitimate station could be flagged. It mirrors today's recomputeWorldOrigin() behavior and is harmless while nothing triggers classification without a global CS — but Commit 3 wires the self-owned triggers, which is exactly where that assumption stops holding. Decide here: skip classification when the global CS is empty (recommended), or require a global CS before gathering. Add a test for the empty-global-CS + mixed-input-CS case.

Commit 4 — CavePage banner (QML)

  • edit CavePage.qml — ErrorHelpArea in the info column bound to currentCave.errorModel, standalone + two LayoutItemProxys so it shows in both wide and narrow layouts.
  • new test-qml/tst_CavePage_outlierWarning.qml — banner visible when the cave carries the warning; hidden otherwise.

Commit 5 — Render-view overlay (QML + validator summary)

  • new cwFixStationValidator::warningMessage() / outlierCount() — the Q_PROPERTYs from §3, summarizing across caves (empty when none), naming the first offending cave. Updated inside revalidate().
  • edit GLTerrainRenderer.qml — top-anchored ErrorHelpBox overlay visible: RootData.region.fixStationValidator.warningMessage !== "", copying the LazClip pattern.
  • new QML test — overlay visible when a cave has an outlier; a link/hint routes the user toward the cave's fix stations.
Each commit is independently green. 1 adds dormant pure code; 2 is a self-contained behavior change; 3 lights up the cave list with no new QML; 4 and 5 are additive QML surfaces. Commit 2 is the only cuttable one.

10. Risks & open decisions

ItemNote
Trigger timingThe validator owns its own connections to each cave's fixStations signals, so it re-runs on a coordinate edit independent of the solve/origin gate. Resolved by design in Commit 3 — no reliance on the line-plot re-firing.
Ownership / lifetimeRegion constructs the validator after geoReference and caves exist; validator holds a raw cwCavingRegion* parent back-pointer (it is a child QObject, dies with the region). It must rewire per-cave connections as caves are added/removed.

Few fixes1–3 fixes → no cluster → detection off (by design). A two-fix survey with one typo won't be caught; acceptable (nothing to compare against).
cwCave child aggregationcwCave's errorModel does not roll up trip children today (wiring commented out). We append directly to the cave list, so no dependence on aggregation.

11. Out of scope (v2)

  • Per-fix-station-row flagging on the Fix Stations subpage. cwFixStationModel has zero error state today; per-row needs a QMap<QUuid, cwErrorModel*> + setParentModel roll-up (the cwSurveyChunk cell pattern). Cave-level attribution in v1 already lights up the list badge, CavePage, and the render overlay — row-level is a refinement.
  • Bad-shot blowups (e.g. a 99999 distance) — a different, harder attribution problem; separate issue.
  • Auto-fix / "did you mean zone 13?" suggestions — future nicety.
  • Reproject primitive on cwGeoReference (post-Commit 2 refactor). The "resolve inputCS → validate → same-CS shortcut → transform into global" logic lives inline in both recomputeWorldOrigin() and the validator; Commit 2 collapses it to a single copy inside the validator, but its natural home is a tested cwGeoReference::reproject(cwGeoPoint, inputCS) → std::optional<cwGeoPoint> on the CS authority itself (blast radius: one method + 1–2 callers). Deferred because the duplication self-resolves in Commit 2; this is a further consolidation, and routing everything through the validator is a defensible alternative.
