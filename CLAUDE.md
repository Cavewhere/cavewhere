## What is CaveWhere

CaveWhere is a Qt 6 / C++ desktop application for cave mapping and 3D visualization. It features 2D-to-3D survey morphing, loop closure detection, Git-based project sync, and LiDAR note processing.

## Build Commands

From a fresh checkout:
```bash
git submodule update --init --recursive
conan profile detect --force
# Pin CMake below 4.0 to avoid wxWidgets compatibility issues:
echo -e "\n[tool_requires]\n!cmake/*: cmake/[>=3 <4]" >> "$(conan config home)/profiles/default"
conan install . -o "&:system_qt=False" --build=missing -of conan_deps
cmake --preset conan-release -DCMAKE_TOOLCHAIN_FILE=conan_deps/conan_toolchain.cmake
cmake --build build/<preset> --target all
```

The build directory name (`build/<preset>`) is generated by CMake from the Qt version and platform, e.g. `build/Qt_6_10_0_for_macOS-Debug`. Use `ls build/` to find the correct name.

Run the application:
```bash
./build/<preset>/CaveWhere
```

Build and run tests:
```bash
cmake --build build/<preset> --target cavewhere-test cavewhere-qml-test
./build/<preset>/cavewhere-test
./build/<preset>/cavewhere-qml-test --platform offscreen
```

Run a single C++ test (Catch2 tag syntax):
```bash
./build/<preset>/cavewhere-test "[TagName]"
```

Run a single QML test:
```bash
./build/<preset>/cavewhere-qml-test --platform offscreen -testall tst_YourTest
```

Always run both suites before pushing.

## Architecture Overview

### Entry Point and Root Singleton

`main.cpp` sets up the QML engine and exposes `cwRootData` as a QML singleton. `cwRootData` is the central hub — it owns and exposes all major subsystems to QML: `cwProject`, `cwCavingRegion`, `cwScrapManager`, `cwLinePlotManager`, `cwSurveyImportManager`, `cwSurveyExportManager`, `cwRemoteServices`, `cwPageSelectionModel`, `cwSettings`, and more.

### Data Model Hierarchy

```
cwCavingRegion (root)
└── cwCave (multiple)
    └── cwTrip (multiple)
        └── cwSurveyChunk (shots)
            ├── cwShot (from/to station, distance/bearing/inclination)
            └── cwNote (2D scanned image)
                └── cwScrap (digitized outlines)
```

### Project Persistence (`cwProject`)

Handles all file I/O and Git integration. Three formats exist:
- **Legacy `.cw`**: JSON + SQLite (auto-converted on load)
- **Bundled `.cw`**: Zip archive (new default for single-file saves)
- **`.cwproj` directory**: Git-backed for collaboration

Serialization uses Protocol Buffers (`cavewhere.proto`). Save/load operations are queued through `cwSaveLoad` to avoid blocking the UI thread.

### Rendering

Uses Qt's RHI (Rendering Hardware Interface). Key classes: `cwRegionSceneManager`, `cwRhiScene`, `cwRhiItemRenderer`. Line plots are handled by `cwLinePlotManager`. 3D math is in the `QMath3d` submodule.

### Survey Import/Export

`cwSurveyImportManager` handles Compass, Survex, and Walls formats via the `dewalls` submodule. `cwSurveyExportManager` outputs Survex. `cwSurveyNetwork` performs loop closure detection.

### Remote/Git Features

`QQuickGit` submodule wraps libgit2. `cwGitHubDeviceAuth` handles OAuth. Credentials are stored via `qtkeychain`. `cwRemoteRepositoryCloner` and `cwRemoteAccountModel` manage remote repository workflows.

### UI Structure

- **Page navigation**: `cwPageSelectionModel` manages multi-page routing
- **Keyword filtering**: `cwKeywordItemModel` / `cwKeywordFilterPipelineModel` for hierarchical search
- **Themes**: All colors go through `Theme.qml` tokens — never hardcode hex values in component QML files

## Coding Conventions

### C++
- 4-space indentation, opening braces on the next line
- `UpperCamelCase` for classes, `m_` prefix for members, `lowerCamelCase` for functions
- Full `const` correctness; use Qt signal/slot patterns with `Q_OBJECT`
- Address Sanitizer is enabled in macOS Debug builds

### QML
- Strict typed properties (no `property var`); lowerCamelCase property names
- Capitalized component filenames (`MyComponent.qml`)
- Use `Theme.qml` for all colors; no hardcoded hex values in component QML
- Prefer `id` selectors over `objectName`; keep bindings simple; avoid deep nesting
- Register modules via `qt_add_qml_module` in CMakeLists.txt
- Use `Loader` for deferred/conditional loading; avoid expensive operations in bindings

### Commits
- Short, capitalized, imperative subjects ("Fix", "Add", "Implement")
- Body: what changed, why, build/test commands run
- Attach screenshots for QML UI changes; link related issues
- Note submodule or dependency changes explicitly

## Dependencies

Managed via Conan: Qt 6.10+, Protobuf 6.32.1, libgit2 1.9.1, OpenSSL 3.5.0.

Key submodules: `QMath3d` (3D math), `dewalls` (survey parser), `QQuickGit` (libgit2 bindings), `asyncfuture`, `monad` (Result<T>), `qtkeychain`, `SignalSpyChecker`, `survex` (desktop only).

System Qt installations require Qt 6.10+. CMake must be pinned below 4.0 in the Conan profile (wxWidgets compatibility).
